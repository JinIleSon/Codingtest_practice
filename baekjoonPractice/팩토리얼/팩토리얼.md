## 문제 설명

정수 `N`이 주어졌을 때 `N!`을 출력한다.

팩토리얼의 정의는 다음과 같다.

```
0! = 1
N! = N × (N-1)!  (N ≥ 1)
```

---

## 입출력

**입력**
- 정수 `N` (0 ≤ N ≤ 12)

**출력**
- `N!` 출력

**예시**

| 입력 | 출력 |
|------|------|
| `10` | `3628800` |
| `0` | `1` |
| `1` | `1` |

## 헷갈렸던 부분

**`0!`과 `1!`의 처리**

`N >= 2`인 경우에만 반복문을 돌리고, `N == 0`과 `N == 1`은 따로 분기 처리했다.  
사실 `1!`은 반복문 범위(`range(2, 2)`)가 아예 실행되지 않기 때문에 별도로 처리할 필요가 없었다.

```python
# 이렇게 분기를 나눴지만
if n >= 2:
    for i in range(2, n+1):
        result *= i
else:
    if n == 0:
        result = 1
    else:       # n == 1
        result = 1  # 사실 이 분기는 불필요
```

`result = 1`로 초기화한 상태에서 반복문을 돌리면, `N == 0`이든 `N == 1`이든 자연스럽게 `1`이 나온다.

**`result`의 초기값**

`result = 0`으로 시작하면 곱셈이 전부 `0`이 된다.  
팩토리얼 계산에서는 반드시 `result = 1`로 초기화해야 한다.

---

## 풀이 접근

### 반복문으로 누적 곱 계산

`1`부터 `N`까지 순서대로 곱해나가면 된다.

```
N! = 1 × 2 × 3 × ... × N
```

`result = 1`로 시작해서 `2`부터 `N`까지 반복하며 곱한다.  
(`1`을 곱하는 건 결과에 영향이 없으므로 `range(2, n+1)`로 시작해도 무방하다.)

```python
result = 1
for i in range(2, n + 1):
    result *= i
```

`N == 0` 또는 `N == 1`일 때는 반복문이 실행되지 않으므로 `result`는 그대로 `1`이 된다.  
별도의 분기 처리 없이 엣지 케이스가 자연스럽게 해결된다.

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

n = int(input().strip())

result = 1

if n >= 2:
    for i in range(2, n+1):
        result *= i
else:
    if n == 0:
        result = 1
    else:
        result = 1

print(result)
```

**리팩토링 버전** — 분기 없이 동일한 결과

```python
import sys

input = sys.stdin.readline

n = int(input().strip())

result = 1
for i in range(2, n + 1):
    result *= i

print(result)
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(N)` |
| 공간 복잡도 | `O(1)` |

- `2`부터 `N`까지 한 번씩 순회하므로 `O(N)`
- 별도의 자료구조 없이 변수 하나로 처리하므로 `O(1)`
- `N ≤ 12`로 제한되어 있어 실질적인 연산 횟수는 매우 적다

## 배운 점

**초기값 설정이 결과를 결정한다**  
곱셈 누적에서 `result = 0`으로 시작하면 모든 결과가 `0`이 된다.  
누적 곱은 항상 `result = 1`로 시작해야 한다는 점을 체감했다.

**불필요한 분기를 줄일 수 있다**  
`N == 0`과 `N == 1`을 별도로 처리했지만, `result = 1` 초기화와 `range(2, n+1)` 조합이 이 두 케이스를 자연스럽게 커버한다.  
엣지 케이스를 억지로 분기하기 전에, 로직 자체가 그 케이스를 포함하는지 먼저 확인하는 습관이 중요하다.

**`range()`의 빈 범위 동작 이해**  
`range(2, 2)`처럼 시작과 끝이 같으면 반복문이 아예 실행되지 않는다.  
이 성질을 활용하면 조건문 없이도 `N == 0`, `N == 1` 같은 케이스를 처리할 수 있다.

---

> 처음에는 `0!`과 `1!`을 따로 처리해야 한다고 생각해서 분기가 늘어났지만,  
> `result = 1` 초기화와 `range`의 특성을 이해하고 나니  
> 조건 없이도 모든 케이스가 자연스럽게 해결된다는 걸 알게 되었다.
