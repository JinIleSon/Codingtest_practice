## 문제 설명

8진수로 이루어진 수가 문자열로 주어졌을 때, 이를 2진수로 변환하여 출력한다.

8진수 한 자리는 2진수 3자리와 정확히 대응된다.

| 8진수 | 2진수 |
|-------|-------|
| 0 | 000 |
| 1 | 001 |
| 2 | 010 |
| 3 | 011 |
| 4 | 100 |
| 5 | 101 |
| 6 | 110 |
| 7 | 111 |

단, 맨 앞자리를 그대로 3자리로 출력하면 불필요한 0이 붙을 수 있다.

```
1 → 001 (❌)  →  1   (✅)
2 → 010 (❌)  →  10  (✅)
3 → 011 (❌)  →  11  (✅)
```

---

## 입출력

**입력**
- 8진수 문자열 1개

**출력**
- 2진수로 변환한 결과

**예시**

| 입력 | 출력 | 설명 |
|------|------|------|
| `314` | `11001100` | 3→`11`, 1→`001`, 4→`100` |
| `1` | `1` | 앞자리 0 없이 출력 |
| `0` | `0` | 그대로 출력 |

---

## 헷갈렸던 부분

**1. 맨 앞자리도 무조건 3자리로 맞추면 안 된다**

처음에는 모든 자리를 동일하게 3비트로 변환하면 된다고 생각했다.  
하지만 앞자리 0이 없는 형태로 출력해야 하므로 **첫 자리만 예외**다.

| 8진수 | 첫 자리 출력 | 나머지 자리 출력 |
|-------|------------|----------------|
| 0 | `0` | `000` |
| 1 | `1` | `001` |
| 2 | `10` | `010` |
| 3 | `11` | `011` |
| 4~7 | 3비트 그대로 | 3비트 그대로 |

4 이상은 3비트 표현의 맨 앞이 `1`이므로, 첫 자리든 아니든 동일하게 출력해도 된다.

**2. `continue`를 잘못 놓으면 값이 누락된다**

`i == 0`일 때 처리 후 `continue`를 성급하게 쓰면, 아래의 4~7 처리 코드까지 건너뛰게 된다.

```python
if i == 0:
    # 0~3 처리
    continue  # ← 여기에 두면 첫 자리가 4~7일 때 누락됨
```

`continue` 위치 하나로 특정 값이 통째로 누락될 수 있다는 점을 확인했다.

**3. `if / elif / if` 구조의 흐름을 정확히 봐야 한다**

이 코드는 앞부분에서 `if ~ elif` 로 0~3을 처리하고, 별도의 `if`로 4~7을 처리하는 구조다.

```
0 ~ 3  →  앞의 if ~ elif 블록에서 처리
4 ~ 7  →  별도 if 블록에서 처리
```

`elif`는 앞 조건이 맞으면 아래를 보지 않지만, 별도의 `if`는 다시 검사한다.  
이 차이를 이해하지 않으면 "왜 어떤 숫자는 여기서 끝나고, 어떤 숫자는 밑으로 내려가지?" 같은 혼동이 생긴다.

---

## 풀이 접근

### ❌ 처음에 놓치기 쉬운 생각

모든 숫자를 3자리 이진수로 바꿔 이어 붙이면 될 것처럼 보인다.

```
3 → 011
1 → 001
4 → 100
→ 011001100  (❌)
```

실제 정답은 `11001100`이다. 앞자리 0을 제거해야 한다.

### ✅ 핵심 아이디어 — 첫 자리와 나머지를 나눠서 처리

문자열을 한 글자씩 순회하면서, **첫 자리는 짧게, 나머지는 3자리로 고정** 출력한다.

```
1. 8진수 문자열을 한 글자씩 본다.
2. 첫 글자라면 앞자리 0이 생기지 않도록 짧게 출력한다.
3. 두 번째 글자부터는 반드시 3자리 이진수로 출력한다.
4. 모든 자리를 이어 붙이면 정답이 된다.
```

굳이 8진수를 정수로 바꾼 뒤 다시 2진수로 계산할 필요가 없다.  
각 자리의 대응 관계가 명확하므로 **문자열을 그대로 순회하며 바로 출력**하는 방식이 더 단순하다.

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

s = list(input().strip())

for i in range(len(s)):
    if i == 0:
        # 맨 앞자리: 앞자리 0이 생기지 않도록 짧게 출력
        if s[i] == '0':
            print('0', end='')
        elif s[i] == '1':
            print('1', end='')
        elif s[i] == '2':
            print('10', end='')
        elif s[i] == '3':
            print('11', end='')
        # i == 0일 때 continue를 여기에 두면 4, 5, 6, 7이 누락됨 — 주의!
    else:
        # 나머지 자리: 반드시 3자리로 출력
        if s[i] == '0':
            print('000', end='')
        elif s[i] == '1':
            print('001', end='')
        elif s[i] == '2':
            print('010', end='')
        elif s[i] == '3':
            print('011', end='')

    # 4~7은 첫 자리든 아니든 3비트 표현이 동일하므로 공통 처리
    if s[i] == '4':
        print('100', end='')
    if s[i] == '5':
        print('101', end='')
    if s[i] == '6':
        print('110', end='')
    if s[i] == '7':
        print('111', end='')
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(N)` |
| 공간 복잡도 | `O(1)` |

> `N` = 8진수 문자열 길이

문자열을 한 번만 순회하고, 각 문자마다 고정된 분기만 수행하므로 전체 시간복잡도는 `O(N)`이다.  
추가 자료구조를 거의 사용하지 않으므로 공간복잡도도 크지 않다.

## 배운 점

**첫 자리 예외 처리를 먼저 확인해야 한다**  
진법 변환이라고 해서 기계적으로 자리수를 맞추면 되는 게 아니었다.  
출력 형식에서 **앞자리 0을 허용하는지**를 먼저 확인하는 습관이 필요하다.

**`continue` 위치 하나로 값이 통째로 누락될 수 있다**  
구현 문제라도 분기 흐름 하나가 결과를 바꿀 수 있다.  
코드를 작성한 뒤 특정 입력값을 머릿속으로 따라가보는 습관이 중요하다.

**`if / elif / if` 구조는 의도적으로 쓰는 패턴이다**  
`elif`는 앞 조건이 맞으면 아래를 보지 않지만, 별도 `if`는 다시 검사한다.  
이 차이를 이해하면 조건 분기를 더 유연하게 설계할 수 있다.

**문자열 순회만으로도 충분히 풀 수 있다**  
8진수를 정수로 변환한 뒤 다시 2진수로 계산하는 방식보다,  
각 자리를 직접 매핑하는 방식이 더 단순하고 직관적이었다.

---

> 단순 진법 변환처럼 보여도, 실제 핵심은 **맨 앞자리 예외 처리**와 **분기 흐름 제어**였다.  
> 특히 `continue` 위치 하나로 일부 값이 누락될 수 있다는 점을 통해,  
> 구현 문제에서도 코드 흐름을 끝까지 따라가는 습관이 중요하다는 걸 배웠다.
