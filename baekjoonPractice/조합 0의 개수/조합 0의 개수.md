## 문제 설명

조합

$$\binom{n}{m} = \frac{n!}{m!(n-m)!}$$

의 값을 계산했을 때, **끝에 붙은 연속된 0의 개수**를 출력한다.

예를 들어,

$$\binom{25}{12} = 5200300$$

이므로 끝자리 0의 개수는 **2개**이다.

---

## 입출력

**입력**
- 정수 `n`, `m` (`0 ≤ m ≤ n ≤ 2,000,000,000`)

**출력**
- $\binom{n}{m}$ 뒤에 붙은 0의 개수

**예시**

| 입력 | 출력 |
|------|------|
| `25 12` | `2` |

---

## 헷갈렸던 부분

### 팩토리얼 문제처럼 5만 세면 되는가?

처음에는 `n!`의 끝자리 0 개수를 구할 때처럼 `5`의 개수만 세면 될 것 같았다.

하지만 이 문제는 단순한 팩토리얼이 아니라 **조합**이므로,

$$\binom{n}{m} = \frac{n!}{m!(n-m)!}$$

형태에서 분모와 약분이 일어난다.  
이 과정에서 `2`와 `5`가 각각 얼마나 남는지를 **모두** 봐야 한다.

즉, 조합의 끝자리 0 개수는 최종적으로 남은 `2`의 개수와 `5`의 개수 중 **더 작은 값**이다.

---

### 왜 `(n-m) // 25` 같은 항도 따로 더해야 하는가?

`5`의 배수는 `5`를 최소 1개 갖지만,  
`25 = 5 × 5`는 `5`를 2개, `125 = 5³`는 `5`를 3개 갖는다.

그래서 단순히 `N // 5`만 하면 부족하고, 다음처럼 5의 거듭제곱 단위로 계속 더해야 한다.

```
N // 5 + N // 25 + N // 125 + ...
```

이렇게 해야 `N!` 안에 들어 있는 5의 **총 개수**를 정확히 셀 수 있다.  
2도 완전히 같은 방식으로 센다.

---

### 직접 조합 값을 구하면 왜 안 되는가?

`n`의 범위가 최대 **2,000,000,000**이기 때문에 `n!`, `m!`, `(n-m)!`을 직접 계산하는 것은 사실상 불가능하다.  
조합 값을 직접 만든 뒤 문자열로 바꿔서 끝의 0을 세는 방식은 숫자 크기 자체가 감당되지 않으므로 사용할 수 없다.

## 풀이 접근

### 핵심 아이디어

끝자리 0은 `10 = 2 × 5`의 쌍에서 생긴다.  
따라서 $\binom{n}{m}$ 안에 남아 있는 **2의 개수**와 **5의 개수**를 각각 구한 뒤, 그중 **더 작은 값**이 정답이 된다.

### 조합에서 소인수 개수 구하기

어떤 소수 $p$에 대해 $\binom{n}{m}$ 안에 들어 있는 $p$의 개수는 다음과 같다.

$$v_p\!\left(\binom{n}{m}\right) = v_p(n!) - v_p(m!) - v_p((n-m)!)$$

여기서 $v_p(N!)$는 다음 공식으로 계산한다.

$$v_p(N!) = \left\lfloor\frac{N}{p}\right\rfloor + \left\lfloor\frac{N}{p^2}\right\rfloor + \left\lfloor\frac{N}{p^3}\right\rfloor + \cdots$$

### 계산 순서

```
1. n! 안의 5 개수 계산
2. m! 안의 5 개수 빼기
3. (n-m)! 안의 5 개수 빼기

4. n! 안의 2 개수 계산
5. m! 안의 2 개수 빼기
6. (n-m)! 안의 2 개수 빼기

7. min(남은 2 개수, 남은 5 개수) 출력
```

### 예시: n=25, m=12

**5의 개수**

$$v_5(25!) = \lfloor25/5\rfloor + \lfloor25/25\rfloor = 5 + 1 = 6$$

$$v_5(12!) = \lfloor12/5\rfloor = 2, \quad v_5(13!) = \lfloor13/5\rfloor = 2$$

$$v_5\!\left(\binom{25}{12}\right) = 6 - 2 - 2 = 2$$

**2의 개수**

$$v_2(25!) = 12 + 6 + 3 + 1 = 22$$

$$v_2(12!) = 6 + 3 + 1 = 10, \quad v_2(13!) = 6 + 3 + 1 = 10$$

$$v_2\!\left(\binom{25}{12}\right) = 22 - 10 - 10 = 2$$

**최종 정답**

$$\min(2,\ 2) = \mathbf{2}$$

## 최종 코드

```python
import sys

input = sys.stdin.readline

n, r = map(int, input().strip().split())

# nCr = n! / (r! * (n-r)!)
# 끝자리 0 = min(소인수 2의 개수, 소인수 5의 개수)

def count_factor(n, r, p):
    """조합 nCr 안에 소인수 p가 몇 개 있는지 계산"""
    power = p
    count = 0
    while n >= power:
        count += n // power
        if r >= power:
            count -= r // power
        if (n - r) >= power:
            count -= (n - r) // power
        power *= p
    return count

five_count = count_factor(n, r, 5)
two_count  = count_factor(n, r, 2)

print(min(five_count, two_count))
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | $O(\log n)$ |
| 공간 복잡도 | $O(1)$ |

- 5에 대한 반복: $5, 25, 125, \ldots$ → $O(\log_5 n)$
- 2에 대한 반복: $2, 4, 8, \ldots$ → $O(\log_2 n)$
- 둘을 합쳐도 전체는 $O(\log n)$ 수준이므로, `n`이 최대 $2 \times 10^9$여도 충분히 빠르다.

## 배운 점

### 조합은 팩토리얼 문제와 다르게 2와 5를 둘 다 봐야 한다

팩토리얼 $n!$ 하나만 다룰 때는 2가 항상 5보다 많아서 5만 세면 됐다.  
하지만 조합은 **분모와 약분**되기 때문에, 최종적으로는 2가 더 부족해질 수도 있다.  
따라서 2와 5를 각각 세고 더 작은 값을 답으로 써야 한다.

### 소인수 개수를 세는 방식은 매우 강력하다

큰 수 자체를 계산하지 않고도, 그 수 안에 특정 소인수가 몇 개 들어 있는지 계산함으로써 끝자리 0 개수 같은 정보를 효율적으로 구할 수 있다.

### $\lfloor N/p \rfloor + \lfloor N/p^2 \rfloor + \cdots$ 의 의미를 이해해야 한다

단순히 $p$의 배수 개수만 세는 것이 아니라, $p^2, p^3$처럼 같은 소인수를 여러 번 가진 수를 **중복 없이 정확히 반영**하기 위한 공식이다.

### 직접 값 계산보다 구조를 보는 것이 중요하다

처음에는 조합 값을 직접 만든 뒤 문자열로 처리하려고 접근할 수 있지만, 이 문제는 수 자체를 구하는 것이 아니라 **"끝자리 0이 몇 개 생기는 구조인가"** 를 보는 문제다.  
큰 수 문제일수록 값을 직접 다루기보다 구조를 먼저 분석하는 습관이 중요하다.

---

> 처음에는 팩토리얼 문제처럼 5만 세면 될 것 같았지만,  
> 조합은 분모와의 약분 때문에 **2도 함께 고려해야 한다**는 점이 핵심이었다.  
> 결국 이 문제는 조합 값을 직접 구하는 문제가 아니라,  
> **조합 안에 남는 소인수 2와 5의 개수를 세는 문제**라는 걸 이해하고 나서 풀 수 있었다.
