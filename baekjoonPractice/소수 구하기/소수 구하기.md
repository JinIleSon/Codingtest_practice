## 문제 설명

두 정수 M, N이 주어졌을 때, M 이상 N 이하의 소수를 모두 출력하는 문제이다.

### 입력

- 첫째 줄에 자연수 M과 N이 공백으로 주어진다.

### 출력

- M 이상 N 이하의 소수를 한 줄에 하나씩 출력한다.

### 입출력 예시

| 입력 | 출력 |
|------|------|
| `3 16` | `3`<br>`5`<br>`7`<br>`11`<br>`13` |

---

## 문제를 이해하며 헷갈렸던 부분

처음에는 소수를 판별할 때, 어떤 수 n에 대해 2부터 n-1 또는 n//2까지 전부 나눠봐야 한다고 생각했다. 하지만 그렇게 하면 구간 안의 모든 수를 하나씩 검사해야 하므로 시간 초과가 발생한다.

이 문제를 풀면서 중요했던 점은 두 가지였다.

- 개별 수의 소수 판별에서는 **√n까지만 보면 충분**하다.
- 하지만 이 문제처럼 **구간 전체의 소수를 구할 때**는, 각 수를 하나씩 판별하는 것보다 에라토스테네스의 체를 사용하는 것이 더 적합하다.

---

## 풀이 접근

### 처음 접근 — 각 수마다 직접 소수 판별

처음에는 M ~ N의 각 수에 대해 소수인지 직접 판별하려고 했다.

- 2부터 √n까지 나눠보기
- 나눠떨어지는 수가 있으면 합성수
- 끝까지 없으면 소수

이 방식은 한 숫자만 볼 때는 맞는 접근이지만, M ~ N의 모든 수를 순회하고 각 수마다 또 2 ~ √n까지 검사하게 되므로 전체적으로 느려진다.

### 왜 √n까지만 보면 되는가

어떤 수 n이 합성수라면 `n = a × b`로 쓸 수 있다. 만약 a와 b가 둘 다 √n보다 크다면 `a × b > √n × √n = n`이 되어버린다. 이는 `a × b = n`이어야 한다는 조건과 모순이다.

즉, **합성수라면 약수 쌍 중 적어도 하나는 반드시 √n 이하에 있어야 한다.** 그래서 2 ~ √n까지만 확인하면 충분하다.

---

## 핵심 아이디어 — 에라토스테네스의 체

> 소수를 하나씩 찾는 것이 아니라,  
> **소수가 아닌 수들을 미리 지워서 남기는 방식**으로 접근해야 했다.

처음에 모든 수를 소수 후보라고 두고, 작은 수부터 순서대로 그 수의 배수를 제거해나간다.

```
2의 배수 제거 → 3의 배수 제거 → 4는 이미 지워짐 → 5의 배수 제거 → ...
```

마지막에 남아 있는 수들이 소수다.

### 풀이 흐름

1. 0 ~ N까지의 수에 대해 소수 여부를 저장할 배열을 만들고 전부 `True`로 둔다.
2. 0, 1은 소수가 아니므로 제외한다.
3. `i = 2`부터 시작해서 `i * i <= N`까지만 확인한다.
4. `i`의 배수들을 모두 `False`로 바꾼다.
5. 마지막에 M ~ N 구간에서 `True`인 값만 출력한다.

---

## 고민했던 부분

### 1. 소수 판별은 자기 자신 전까지 다 확인해야 한다고 생각했다

처음에는 2부터 자기 자신 직전까지 모두 나눠봐야 한다고 느꼈다. 하지만 합성수라면 약수 쌍 중 하나는 반드시 √n 이하에 있으므로, 자기 자신까지 확인하는 것은 의미가 없었다.

### 2. n//2까지만 보면 되는 줄 알았다

처음에는 절반까지만 보면 충분하다고 생각했지만, 약수는 항상 쌍으로 존재하고 그 쌍의 경계가 √n이기 때문에 절반이 아니라 **√n까지만** 보면 된다.

### 3. √n까지만 보면 된다는 것이 공식처럼 느껴졌다

처음에는 외워야 하는 규칙처럼 느껴졌지만, 사실은 곱셈의 성질에서 자연스럽게 나오는 결과였다.

- `√n × √n = n`
- 둘 다 √n보다 크면 곱이 n보다 커짐
- 따라서 약수 쌍이 있다면 둘 다 √n보다 클 수 없음

이 원리를 이해하고 나니 왜 루트까지만 보면 되는지 납득할 수 있었다.

### 4. 개별 소수 판별과 구간 전체 소수 구하기는 다르다는 점이 헷갈렸다

| 상황 | 방법 |
|------|------|
| 한 숫자 소수 판별 | 2 ~ √n까지 직접 나눠보기 |
| 구간 전체 소수 구하기 | 에라토스테네스의 체 |

한 숫자 판별 최적화와 범위 전체 처리 최적화는 다른 문제였다.

### 5. 체에서도 루트 개념이 아예 없어지는 줄 알았다

에라토스테네스의 체를 쓰면 루트 개념이 필요 없다고 생각하기 쉽지만, 개별 숫자마다 √n 비교는 없어질 뿐 전체 범위에 대해 **`i * i <= N`까지만 보면 충분**하다는 식으로 루트 개념은 여전히 남아 있다.

### 6. 0과 1도 처음에는 소수 후보 배열에 들어간다는 점을 놓치기 쉬웠다

처음에 전부 `True`로 두고 시작하면 0, 1도 같이 포함된다. 따라서 출력하거나 배열 생성 직후에 이 둘을 제외해야 한다.

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

n1, n2 = map(int, input().strip().split())
numberList = []

# 0부터 n2까지 소수 후보로 둔다.
for i in range(0, n2 + 1):
    numberList.append(True)

i = 2
while i * i <= n2:
    j = 2
    # i의 배수들을 False 처리
    while i * j <= n2:
        if numberList[i * j] == True:
            numberList[i * j] = False
        j += 1
    i += 1

# n1부터 n2까지의 범위에서 소수만 출력
for i in range(n1, n2 + 1):
    if i == 0 or i == 1:
        continue
    if numberList[i] == True:
        print(i)
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(N log log N) |
| 공간 복잡도 | O(N) |

에라토스테네스의 체는 각 수를 하나씩 개별 판별하는 방식보다 훨씬 효율적으로 합성수를 제거할 수 있다. 배수 제거가 반복되지만 전체적으로는 매우 빠르게 동작하여 이 문제의 범위에 적합하다.

---

## 보완할 수 있는 점

### 1. 0, 1을 미리 False 처리할 수 있다

지금은 출력할 때 0, 1을 건너뛰고 있다. 배열 생성 직후 다음처럼 처리하면 더 명확하다.

```python
numberList[0] = False
numberList[1] = False
```

### 2. 이미 지워진 수의 배수는 다시 처리하지 않아도 된다

현재 코드는 `i = 4, 6, 8`처럼 이미 합성수인 경우에도 그 배수를 다시 지우고 있다. 틀리진 않지만 비효율적이므로, `numberList[i] == True`일 때만 배수를 지우도록 개선할 수 있다.

### 3. 배수 제거를 i * i부터 시작할 수 있다

`5 * 2 = 10`, `5 * 3 = 15`는 이미 더 작은 수의 단계에서 처리되었을 가능성이 높다. 일반적인 체 구현에서는 `j = i * i`부터 시작한다.

---

## 이번 문제에서 얻은 것

**개별 판별과 전체 범위 처리의 차이** — 한 숫자의 소수 여부를 빠르게 판별하는 방법과, 구간 전체의 소수를 효율적으로 구하는 방법은 다를 수 있다는 점을 알게 되었다.

**루트 개념을 외우는 것이 아니라 이해하게 됨** — √n까지만 보면 되는 이유를 단순 암기가 아니라 곱셈 구조로 이해할 수 있게 되었다.

**에라토스테네스의 체의 핵심 감각** — 소수를 찾는 것이 아니라, 소수가 아닌 수를 지워서 남긴다는 발상을 익힐 수 있었다.

---

## 한 줄 회고

소수를 하나씩 판별하는 방식에서 자꾸 막혔는데, 약수의 쌍과 루트 개념을 이해한 뒤 에라토스테네스의 체로 시점을 바꾸면서 구간 전체 소수를 훨씬 효율적으로 구할 수 있다는 점을 배울 수 있었다.
