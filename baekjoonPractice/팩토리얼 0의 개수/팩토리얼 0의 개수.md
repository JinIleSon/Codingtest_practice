## 문제 설명

`N!`의 결과에서 **끝에 붙은 연속된 0의 개수**를 출력한다.

예를 들어 `10! = 3628800`이므로 뒤에서부터 `00` → 0이 **2개**.

---

## 입출력

**입력**
- 정수 `N` (0 ≤ N ≤ 500)

**출력**
- `N!` 뒤에 붙은 0의 개수

**예시**

| 입력 | 출력 | 참고 |
|------|------|------|
| `10` | `2` | `10! = 3628800` |
| `3` | `0` | `3! = 6` |
| `0` | `0` | `0! = 1` |

---

## 헷갈렸던 부분

**0의 개수를 세는 방향**

끝에 붙은 0을 세야 하므로, 문자열의 **뒤에서부터** 순회해야 한다.  
앞에서부터 세면 중간에 있는 0까지 포함되어 버린다.

```python
# 올바른 방향: 뒤에서 앞으로
for i in range(len(result)-1, 0, -1):
    if result[i] == "0":
        count += 1
    else:
        break  # 0이 아닌 숫자를 만나면 즉시 중단
```

**`range`의 끝 범위**

`range(len(result)-1, 0, -1)`은 인덱스 `1`까지만 순회한다 (인덱스 `0`은 포함 안 됨).  
`N!`의 결과가 `"0"`만으로 이루어질 수는 없으므로 실질적인 문제는 없지만,  
의도한 범위인지 한 번 더 확인하는 습관이 필요하다.

**`n == 1`의 별도 처리 여부**

`else: result = 1` 분기에 `n == 0`과 `n == 1`이 모두 포함된다.  
`1! = 1`은 반복문 `range(2, 2)`가 실행되지 않으므로 사실 분기 없이도 처리된다.

## 풀이 접근

### 이 코드의 방식 — 팩토리얼 직접 계산 후 문자열 탐색

`N!`을 직접 계산한 뒤, 결과를 문자열로 변환하고 뒤에서부터 `'0'`을 센다.

```
1. N! 계산
2. str(N!)로 변환
3. 뒤에서부터 순회
4. '0'이면 count += 1, 아니면 break
```

파이썬은 **임의 정밀도 정수**를 지원하기 때문에 `N = 500`이어도 오버플로우 없이 계산된다.  
`500! = 약 10^1134`로 매우 큰 수지만, 파이썬에서는 그대로 다룰 수 있다.

### 수학적 접근 — 5의 배수 세기 (더 효율적인 방법)

끝자리 0은 `10 = 2 × 5`의 쌍에서 나온다.  
`N!`에는 `5`의 인수보다 `2`의 인수가 항상 더 많으므로, **5의 인수 개수**만 세면 된다.

```
count = N//5 + N//25 + N//125 + N//625 + ...
```

| N | N//5 | N//25 | N//125 | 합계 |
|---|------|-------|--------|------|
| 10 | 2 | 0 | 0 | **2** |
| 25 | 5 | 1 | 0 | **6** |
| 100 | 20 | 4 | 0 | **24** |
| 500 | 100 | 20 | 4 | **124** |

이 방식은 팩토리얼을 직접 계산하지 않아도 되므로 훨씬 빠르다.

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

n = int(input().strip())

result = 1
count = 0

if n >= 2:
    for i in range(2, n+1):
        result *= i
else:
    result = 1

result = str(result)

for i in range(len(result)-1, 0, -1):
    if result[i] == "0":
        count += 1
    else:
        break

print(count)
```

**수학적 풀이 버전** — 팩토리얼 계산 없이 O(log N)

```python
import sys

input = sys.stdin.readline

n = int(input().strip())

count = 0
power = 5

while power <= n:
    count += n // power
    power *= 5

print(count)
```

## 복잡도 분석

### 이 코드 (팩토리얼 직접 계산)

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(N × D)` |
| 공간 복잡도 | `O(D)` |

> `D` = `N!`의 자릿수 (N=500일 때 약 1135자리)

- 팩토리얼 계산: `O(N)` 반복이지만, 각 단계에서 큰 수 곱셈 비용이 추가됨
- 문자열 변환 및 탐색: `O(D)`
- N이 작은 이 문제에서는 충분히 통과 가능

### 수학적 풀이

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(log N)` |
| 공간 복잡도 | `O(1)` |

## 배운 점

**끝자리 0은 `2 × 5` 쌍에서 나온다**  
0이 붙는 이유를 수학적으로 따지면, `10`의 인수 쌍 즉 `2 × 5`가 몇 번 나오는지의 문제다.  
`N!`에서 `2`는 항상 `5`보다 많으므로, `5`의 인수 개수만 세면 끝자리 0의 개수가 된다.

**파이썬은 큰 수를 그대로 다룰 수 있다**  
`500!`처럼 천 자리가 넘는 수도 오버플로우 없이 계산된다.  
다른 언어라면 BigInteger 같은 별도 타입이 필요하지만, 파이썬에서는 그냥 `int`로 처리 가능하다.

**문자열 탐색 방향에 주의해야 한다**  
끝자리 0을 세려면 뒤에서부터 순회해야 하고, 0이 아닌 숫자를 만나는 순간 `break`해야 한다.  
앞에서 세거나 `break` 없이 전체를 순회하면 중간에 있는 0까지 포함되어 잘못된 결과가 나온다.

**직접 계산과 수학적 접근의 선택**  
이 문제는 팩토리얼을 직접 계산해도 통과되지만, `N`이 더 커진다면 문자열 처리 비용이 급격히 증가한다.  
`5`의 인수를 세는 방식은 `O(log N)`으로 훨씬 효율적이며, 이런 수학적 관찰을 먼저 시도하는 습관이 중요하다.

---

> 처음에는 팩토리얼을 그대로 계산하고 문자열로 세면 된다고 생각했지만,  
> 끝자리 0이 생기는 원리를 이해하고 나면  
> 팩토리얼 자체를 계산할 필요 없이 훨씬 단순하게 풀 수 있다는 걸 알게 되었다.
