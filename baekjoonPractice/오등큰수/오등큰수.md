## 문제 설명

크기가 N인 수열 A가 주어졌을 때, 각 원소 A[i]에 대해 오른쪽에 있으면서 **등장 횟수가 더 큰 수 중 가장 왼쪽에 있는 수**를 구하는 문제이다.

이 수를 **오등큰수(NGF, Next Greater Frequency)** 라고 한다. 만약 오른쪽에 자신보다 등장 횟수가 더 큰 수가 없다면 `-1`을 출력한다.

### 입력

- 첫째 줄에 수열의 크기 N
- 둘째 줄에 수열 A₁ A₂ ... Aₙ

### 출력

- 총 N개의 수를 공백으로 구분하여 출력
- 각 위치의 오등큰수를 순서대로 출력

### 입출력 예시

| 입력 | 출력 |
|------|------|
| `7`<br>`1 1 2 3 4 2 1` | `-1 -1 1 2 2 1 -1` |

---

## 문제를 이해하며 헷갈렸던 부분

처음에는 이름이 오큰수와 비슷해서 단순히 **현재 값보다 큰 수를 찾는 문제**처럼 느껴졌다. 그런데 이 문제는 숫자 크기를 비교하는 것이 아니라, **그 숫자가 전체에서 몇 번 나왔는지**를 기준으로 비교하는 문제였다.

예를 들어 수열이 `1 1 2 3 4 2 1`이라면 각 숫자의 등장 횟수는 다음과 같다.

| 숫자 | 등장 횟수 |
|------|-----------|
| `1` | 3번 |
| `2` | 2번 |
| `3` | 1번 |
| `4` | 1번 |

여기서 `2`의 오등큰수는 오른쪽에 있는 값 중 **등장 횟수가 2보다 큰 값**을 찾아야 하므로 `1`이 된다. 처음에는 "제일 많이 나온 횟수는 3이니까 답이 3인가?" 하고 헷갈렸는데, 실제로 출력해야 하는 것은 **횟수 자체가 아니라 그 숫자 값**이었다.

즉, **비교 기준은 등장 횟수**이고, **결과로 출력하는 것은 숫자 값**이라는 점을 분리해서 이해해야 했다.

---

## 풀이 접근

### 처음 접근 — 각 원소마다 오른쪽을 확인

처음에는 각 원소마다 오른쪽을 끝까지 보면서 등장 횟수가 더 큰 값이 나오는지 확인하는 방식으로 생각했다.

- 현재 인덱스 `i`를 기준으로
- `j = i + 1`부터 끝까지 순회하면서
- `freq[A[j]] > freq[A[i]]` 인 첫 번째 값을 찾고
- 없으면 `-1`

이 방식도 결국 **각 원소마다 오른쪽을 다시 확인**하게 되므로 최악의 경우 **O(N²)** 이 되어 비효율적이다.

### 등장 횟수는 먼저 구해야 한다

"등장 횟수를 비교해야 하는데 그걸 매번 세면 또 이중 반복문 아닌가?" 하고 막혔던 부분이 있었다. 하지만 실제로는 **등장 횟수 자체는 한 번만 미리 구해두면 된다.**

1. 먼저 수열 전체를 한 번 돌면서 각 숫자의 등장 횟수를 저장
2. 그다음 순회에서는 저장된 횟수를 꺼내 비교

이 흐름으로 가면 등장 횟수 계산 때문에 이중 반복문이 되는 것은 아니었다.

---

## 핵심 아이디어 — 스택으로 미해결 인덱스 관리

> 현재 원소의 답을 찾으러 가는 것이 아니라,  
> **현재 값이 이전의 미해결 원소들의 오등큰수가 되는지 확인**하는 방식으로 생각해야 했다.

오큰수와 큰 틀은 비슷하지만, 비교 기준만 다르다.

| 문제 | 비교 기준 |
|------|-----------|
| 오큰수 | 값의 크기 |
| 오등큰수 | 등장 횟수 |

스택에는 **아직 오등큰수가 정해지지 않은 인덱스**를 넣어둔다. 현재 인덱스 `i`를 보면서 스택 맨 위 인덱스가 가리키는 값의 등장 횟수보다 현재 값의 등장 횟수가 더 크다면, 현재 값이 그 인덱스의 오등큰수가 된다. 조건이 맞는 동안 계속 pop 하면서 결과를 채운다.

### 풀이 흐름

1. 수열 전체를 한 번 순회하며 각 숫자의 등장 횟수를 센다.
2. 수열을 다시 왼쪽부터 오른쪽으로 순회한다.
3. 현재 값의 등장 횟수가 스택 맨 위 인덱스의 값보다 크다면, 현재 값이 그 인덱스의 오등큰수이므로 결과 배열에 저장하고 pop 한다.
4. 조건이 맞지 않을 때까지 반복한다.
5. 현재 인덱스는 아직 자신의 오등큰수를 모르는 상태이므로 스택에 넣는다.
6. 순회가 끝난 뒤에도 스택에 남아 있는 인덱스들은 끝까지 오등큰수를 찾지 못한 것이므로 `-1`로 처리한다.

## 고민했던 부분

### 1. 오큰수처럼 보이는데 비교 기준이 달라서 헷갈렸다

문제 이름이 비슷해서 처음에는 값의 크기를 비교한다고 느껴졌지만, 실제로는 **숫자의 크기**가 아니라 **등장 횟수의 크기**를 비교해야 했다. `4`가 `2`보다 크다고 해서 답이 되는 것이 아니라, `4`의 등장 횟수가 `2`의 등장 횟수보다 커야 답이 된다.

### 2. 결과에 들어가는 게 횟수인지 숫자인지 헷갈렸다

예제에서 `2`의 오등큰수가 `1`이니까 출력값이 횟수인 `3`이어야 하는 것 아닌가 하고 잠깐 헷갈렸다. 하지만 출력하는 것은 등장 횟수가 아니라 **그 조건을 만족하는 실제 숫자 값**이다.

- 비교는 `freq`
- 저장은 `number[i]`

### 3. 등장 횟수를 구하는 것부터 이중 반복문처럼 느껴졌다

등장 횟수를 비교하려면 매번 다시 세야 할 것 같아서 걱정됐다. 하지만 각 숫자의 횟수를 한 번만 세어두면 이후에는 배열에서 바로 꺼내 쓰면 되므로 등장 횟수 계산 자체는 O(N)으로 처리할 수 있었다.

### 4. 스택에 값과 인덱스 중 무엇을 넣어야 하는지 헷갈렸다

처음에는 스택에 실제 값 `number[i]`를 넣고 처리하려고 했다. 그런데 결과를 채워야 하는 위치는 `result[i]` 형태의 **인덱스 기반**이고, 같은 값이 여러 번 나올 수도 있기 때문에 값만 넣으면 어느 위치의 답을 채워야 하는지 알 수 없었다. 그래서 스택에는 **인덱스**를 넣어야 한다.

- 비교할 때는 `freq[number[인덱스]]`
- 스택에는 `인덱스`
- 결과에는 `number[i]`

### 5. 값 / 인덱스 / 등장 횟수가 같이 얽혀서 더 헷갈렸다

이 문제는 단순히 값만 다루는 게 아니라 세 가지가 동시에 등장해서 꼬이기 쉬웠다.

- `number[i]` = 실제 값
- `i` = 위치
- `freq[number[i]]` = 그 값의 등장 횟수

이 세 줄로 분리해서 생각하니 그제야 구조가 보이기 시작했다.

### 6. 최대 등장 횟수인 값만 보면 되는 줄 알았다

"가장 많이 나온 값이 기준이니까 최대 등장 횟수를 저장해두고 그 값만 보면 되는 것 아닌가?" 하고 생각했다. 하지만 오등큰수는 단순히 최대 등장 횟수인 값을 찾는 문제가 아니라, **현재 위치의 오른쪽에서 처음으로 등장 횟수가 더 큰 값을 찾는 문제**다. 반드시 오른쪽에 있는지, 그리고 가장 먼저 만나는지까지 함께 봐야 했다.

## 최종 코드

```python
import sys

input = sys.stdin.readline

N = int(input().strip())
number = list(map(int, input().strip().split()))

stack = []
result = [-1] * N

# 문제 조건상 수의 범위에 맞게 빈도 배열 생성
freq = [0] * 1000001

# 각 숫자의 등장 횟수 계산
for i in range(N):
    freq[number[i]] += 1

# 오등큰수 처리
for i in range(N):
    while len(stack) > 0 and freq[number[stack[-1]]] < freq[number[i]]:
        result[stack[-1]] = number[i]
        stack.pop()
    stack.append(i)

print(*result)
```

---

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

겉으로 보면 `for`문 안에 `while`문이 있어서 이중 반복처럼 보일 수 있다. 하지만 각 인덱스는 스택에 한 번 들어가고 최대 한 번 pop 된다. 또 등장 횟수 계산도 전체를 한 번만 순회하면 되므로, 전체 시간 복잡도는 **O(N)** 이다.

## 이번 문제에서 얻은 것

**오큰수와 비슷한 구조를 다른 기준으로 확장하는 감각** — 스택 문제의 핵심 구조는 유지되지만, 비교 기준이 값에서 등장 횟수로 바뀔 수 있다는 점을 체감했다.

**비교 기준과 저장 대상은 다를 수 있다는 점** — 비교는 등장 횟수로 하지만 결과에는 숫자 값을 넣어야 했고, 스택에는 인덱스를 넣어야 했다. 무엇을 기준으로 비교하고 무엇을 저장하는지 분리해서 생각하는 연습이 되었다.

**한 번 더 추상화해서 보는 연습** — 오큰수를 그대로 외워서 적용하는 것이 아니라, "현재 값이 이전 미해결 원소의 답이 되는가"라는 구조 자체를 이해해야 비슷한 문제에 응용할 수 있다는 점을 느꼈다.

## 한 줄 회고

오큰수와 비슷해 보여도 비교 기준이 값이 아니라 등장 횟수로 바뀌면서 훨씬 더 헷갈렸지만, 값·인덱스·등장 횟수를 분리해서 생각해야 한다는 점을 제대로 배울 수 있었다.
