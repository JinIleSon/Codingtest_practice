## 📌 문제 설명

두 개의 자연수 `A`, `B`가 주어졌을 때, **최대공약수(GCD)** 와 **최소공배수(LCM)** 를 구하는 문제.

### 입력 / 출력

```
입력: 24 18
---
출력:
6       ← 최대공약수
72      ← 최소공배수
```

---

## 💡 풀이 접근

### 처음 접근 — 소인수분해로 공통 부분 찾기

두 수를 각각 소인수분해한 뒤, 공통 소인수를 기준으로 GCD / LCM을 구하는 방식으로 접근했다.

예를 들어 `24`와 `18`이라면:

```
24 = 2 × 2 × 2 × 3
18 = 2 × 3 × 3

공통 소인수: 2, 3  →  GCD = 6
전체 소인수 합집합: 2 × 2 × 2 × 3 × 3  →  LCM = 72
```

개념적으로는 맞는 방향이었지만, **리스트를 순회하면서 pop으로 삭제하는 과정**에서 인덱스가 꼬이는 문제가 반복해서 발생했다.

---

## 🚧 구현 중 막혔던 부분

### 1. 소인수분해 종료 시점

한 소인수로 나누다 보면 다음 소인수를 어디서부터 탐색할지가 불명확했다. 아래 세 가지를 직접 처리해야 했다.

- 현재 약수로 계속 나눌 수 있는지 확인
- 더 이상 나눠지지 않으면 다음 약수 탐색 시작점 설정
- 마지막에 `1`이 아닌 잔여값 처리

### 2. 리스트 수정 중 순회 문제

`pop(j)` 호출 후 인덱스가 당겨지는 문제:

| 상황 | 문제 |
|------|------|
| pop 후 `j += 1` | 당겨진 원소를 건너뜀 |
| pop 후 `continue` | 같은 인덱스를 반복 확인 |
| 같은 소인수 여러 번 등장 | 1:1 대응이 꼬임 |

> **배운 점**: 리스트를 수정하면서 순회하는 코드는 항상 조심해야 한다.

### 3. GCD와 LCM 역할 혼동

- 공통 소인수만 곱한 값 → **GCD**
- 공통 + 남은 소인수 전부 곱한 값 → **LCM**

구현 중 변수명과 실제 역할이 뒤섞이는 경우가 많았다.

## 📝 최종 코드

```python
import sys

input = sys.stdin.readline
n1, n2 = map(int, input().strip().split())
n = 2

# 더 큰 수를 n1으로 정렬
if n1 < n2:
    n1, n2 = n2, n1

n1List = []
n2List = []
LCM = 1
GCD = 1

# n1 소인수분해
for i in range(2, n1 + 1):
    if n1 % i == 0:
        n = i
        break

dummy = 0
while n1 % n == 0:
    n1List.append(n)
    n1 //= n
    if n1 % n != 0:
        dummy = n
        for i in range(dummy, n1 + 1):
            if n1 % i == 0:
                n = i
                break
if n1 != 1:
    n1List.append(n1)

# n2 소인수분해
for i in range(2, n2 + 1):
    if n2 % i == 0:
        n = i
        break

dummy = 0
while n2 % n == 0:
    n2List.append(n)
    n2 //= n
    if n2 % n != 0:
        dummy = n
        for i in range(dummy, n2 + 1):
            if n2 % i == 0:
                n = i
                break
if n2 != 1:
    n2List.append(n2)

# 공통 소인수 → GCD
for i in range(len(n1List)):
    j = 0
    while j < len(n2List):
        if n1List[i] == n2List[j]:
            GCD *= n2List.pop(j)
            break
        j += 1

print(GCD)

# 남은 소인수 전부 곱하기 → LCM
for i in n1List:
    LCM *= i
for i in n2List:
    LCM *= i

print(LCM)
```

## 📊 복잡도 분석

| 항목 | 내용 |
|------|------|
| 시간 복잡도 | 소인수 탐색 + 리스트 비교 반복으로 비효율적 |

> 이 풀이는 유클리드 호제법을 사용하는 정석 풀이보다 비효율적이다. 입력이 커질수록 불리하다.

## 이번 문제에서 얻은 것

**1. 소인수 관점에서 GCD / LCM 구조를 직접 이해했다**
공식을 외워 쓰는 것이 아니라, 소인수분해를 통해 두 값의 의미를 구체적으로 파악할 수 있었다.

**2. 리스트 수정 중 순회가 얼마나 위험한지 체감했다**
`pop()` 은 단순해 보이지만, 반복문 안에서 사용하면 인덱스가 꼬이기 쉽다.

**3. 아이디어와 구현은 별개의 능력이다**
큰 흐름은 빠르게 잡았지만, 작은 디테일 하나 때문에 오답이 나는 경우가 많았다. 문제를 이해하는 것과 오류 없이 코드를 작성하는 것은 다른 스킬이다.

**4. 정석 풀이가 왜 정석인지 이해됐다**
우회 방법을 직접 구현해보고 나니, 유클리드 호제법이 왜 단순하고 안정적인지를 단순히 외워야 할 공식이 아닌 필연적인 결론으로 받아들일 수 있었다.
