## 문제 설명

코니는 매일 다른 옷을 조합해서 입는다.  
각 종류별로 최대 1가지 의상만 착용할 수 있고, 하루에 최소 1개는 반드시 입어야 한다.  
의상 목록이 주어질 때, **가능한 서로 다른 옷 조합의 수**를 구하는 것이 목표다.

| clothes | return |
|---------|--------|
| `[["yellow_hat","headgear"],["blue_sunglasses","eyewear"],["green_turban","headgear"]]` | 5 |
| `[["crow_mask","face"],["blue_sunglasses","face"],["smoky_makeup","face"]]` | 3 |

---

## 입출력

**입력**

- `clothes`: `[의상 이름, 의상 종류]` 형태의 2차원 배열

**출력**

- 서로 다른 옷 조합의 수

**제한사항**

- 의상의 수는 1개 이상 30개 이하
- 같은 이름의 의상은 존재하지 않음
- 모든 문자열은 알파벳 소문자 또는 `_`로만 이루어짐

---

## 헷갈렸던 부분

### 1. 중첩 딕셔너리가 필요하다고 생각했다

처음에는 종류별 의상 이름을 모두 저장해야 한다고 생각했다.  
예를 들어 `{"headgear": {"yellow_hat", "green_turban"}}` 같은 구조로 접근했는데,  
실제로는 **종류별 개수만** 알면 충분하다.

```python
# 불필요한 중첩 딕셔너리
{"headgear": {"yellow_hat", "green_turban"}, "eyewear": {"blue_sunglasses"}}

# 필요한 건 이것만
{"headgear": 2, "eyewear": 1}
```

### 2. 경우의 수를 직접 세려고 했다

어떤 의상을 입고 어떤 걸 빼는지 직접 나열하려 하면 매우 복잡해진다.  
하지만 **"안 입는 경우"를 선택지 하나로 포함**하면 수식 하나로 끝난다.

```
(종류A 개수 + 1) × (종류B 개수 + 1) × ... - 1
```

각 종류에서 "+1"은 해당 종류를 아예 안 입는 경우를 뜻하고,  
마지막 "-1"은 모든 종류를 다 안 입는 경우(= 아무것도 안 입는 경우)를 제외하는 것이다.

---

## 풀이 접근

### ❌ 처음에 놓치기 쉬운 생각

의상 이름을 종류별로 저장하고, 조합을 직접 만들어야 한다고 느낄 수 있다.  
하지만 이 문제는 조합을 실제로 구성할 필요가 없다.

### ✅ 핵심 아이디어 — "안 입는 경우"를 포함해 곱셈으로 해결

각 종류별로 선택지를 `(의상 개수 + 1)`로 잡는다.  
`+1`은 해당 종류를 안 입는 경우다.

```
정답 = (종류A 수 + 1) × (종류B 수 + 1) × ... - 1
```

**예시 1** — headgear 2개, eyewear 1개

```
(2 + 1) × (1 + 1) - 1
= 3 × 2 - 1
= 5
```

**예시 2** — face 3개 (종류가 1개뿐)

```
(3 + 1) - 1
= 3
```

## 최종 코드

```python
def solution(clothes):
    clothesDict = {}

    # 종류별 개수만 세면 되니 중첩 딕셔너리 불필요
    for i in range(len(clothes)):
        clothesDict[clothes[i][1]] = clothesDict.get(clothes[i][1], 0) + 1

    result = 1
    for key, value in clothesDict.items():
        result *= (value + 1)
    result -= 1

    return result
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(n) |
| 공간 복잡도 | O(k) |

`n` = 의상의 수, `k` = 의상 종류의 수 (k ≤ n)

`clothes`를 한 번 순회해 종류별 개수를 세므로 O(n),  
이후 종류 수만큼 곱셈을 반복하므로 O(k)가 추가된다.

## 배운 점

- **이 문제는 조합 문제가 아니라 경우의 수 수식 문제였다.**  
  "안 입는 경우"를 선택지에 포함하는 순간, 복잡한 조합 탐색이 단순 곱셈으로 바뀐다.

- **딕셔너리에 값을 읽을 때는 `get()`이 필요하지만, 할당할 때는 필요 없다.**  
  존재하지 않는 키를 읽으려 하면 `KeyError`가 발생하므로 읽기에서만 `get()`을 사용한다.

  ```python
  # 읽기: get() 사용 (키 없을 때 기본값 반환)
  clothesDict.get(key, 0)

  # 쓰기: 그냥 할당
  clothesDict[key] = value
  ```

- **막혔을 때는 "내가 지금 뭘 가정하고 있지?"를 스스로 물어보는 게 중요하다.**  
  이번 문제에서는 "구조를 복잡하게 저장해야 한다"는 가정이 숨어 있었는데,  
  실제로는 개수만 있으면 충분했다.

- **문제를 수식으로 바꾸는 습관이 중요하다.**  
  `정답 = 각 종류별 (개수 + 1)의 곱 - 1`로 정리되는 순간 거의 끝이다.

> "안 입는 경우"를 선택지 하나로 추가하면 단순 곱셈 문제로 바뀐다.  
> 복잡하게 느껴지는 문제도 핵심 조건을 수식으로 바꾸면 코드 몇 줄로 풀린다.
