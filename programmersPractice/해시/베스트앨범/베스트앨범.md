## 문제 설명

노래의 장르 배열 `genres`와 재생 횟수 배열 `plays`가 주어진다.  
베스트 앨범에 수록할 노래의 고유 번호를 순서대로 반환해야 한다.

수록 기준은 다음과 같다.

1. 속한 노래가 많이 재생된 **장르**를 먼저 수록한다.
2. 장르 내에서 **많이 재생된 노래**를 먼저 수록한다.
3. 재생 횟수가 같은 노래 중에서는 **고유 번호가 낮은 노래**를 먼저 수록한다.

| genres | plays | return |
|--------|-------|--------|
| `["classic", "pop", "classic", "classic", "pop"]` | `[500, 600, 150, 800, 2500]` | `[4, 1, 3, 0]` |

---

## 입출력

**입력**
- `genres` : 노래의 장르를 나타내는 문자열 배열 (1 이상 10,000 이하)
- `plays` : 노래별 재생 횟수 배열 (`genres`와 길이 동일)

**출력**
- 베스트 앨범에 들어갈 노래의 고유 번호 배열

**제한사항**
- `genres[i]`는 고유번호 `i`인 노래의 장르
- 장르 종류는 100개 미만
- 장르에 속한 곡이 하나라면, 하나의 곡만 선택
- 모든 장르는 재생된 횟수가 다름

---

## 헷갈렸던 부분

**1. 문제 이해: 장르"별"로 최대 2곡씩이라는 것**

처음에는 전체에서 가장 많이 재생된 상위 2곡만 뽑는다고 오해했다.  
실제로는 **모든 장르를 순회하면서 각 장르마다 최대 2곡씩** 뽑아야 한다.  
장르가 N개라면 최대 N×2곡이 결과에 포함될 수 있다.

**2. 재생수가 같을 때 고유 번호 처리**

단순히 재생수 기준으로만 정렬하면, 동점일 때 고유 번호가 낮은 노래가 먼저 온다는 보장이 없다.  
`(재생수 내림차순, 고유번호 오름차순)` 두 가지 기준을 동시에 적용해야 한다.

**3. `(재생수, 인덱스)` 튜플로 묶어야 한다**

재생수만 저장하면 정렬 후 원래 고유 번호를 알 수 없다.  
처음부터 `(plays[i], i)` 형태의 튜플로 함께 가지고 다녀야 한다.

**4. `len(sumDict[0]) == 1` 함정**

장르가 1개일 때 노래 수를 확인하려고 `len(sumDict[0])`을 썼는데,  
`sumDict`는 장르 이름 문자열의 리스트이므로 `sumDict[0]`은 `'classic'` 같은 문자열이다.  
`len('classic')`은 문자열 길이 7이 반환되어 의도와 전혀 다르게 동작한다.  
노래 수는 반드시 `len(songDict[sumDict[0]])`으로 확인해야 한다.

## 풀이 접근

### 핵심 아이디어 — 두 단계 정렬

이 문제는 정렬 기준이 두 겹으로 존재한다.

```
1. 장르 단위: 장르별 총 재생수 내림차순
2. 곡 단위: 장르 내에서 재생수 내림차순, 동점이면 고유번호 오름차순
```

**Step 1.** 장르별로 `(재생수, 고유번호)` 튜플을 모은다.

```python
songDict = {}
for i in range(len(genres)):
    songDict.setdefault(genres[i], []).append((plays[i], i))
```

`setdefault`는 키가 없으면 빈 리스트로 초기화하고, 있으면 기존 리스트를 그대로 반환한다.  
`if/else` 분기 없이 한 줄로 처리할 수 있다.

**Step 2.** 각 장르의 곡 목록을 정렬하고 장르별 총합을 구한다.

```python
songDict[key].sort(key=lambda x: (-x[0], x[1]))
```

`key=lambda x: (-x[0], x[1])`은 비교용 튜플을 새로 만들어 정렬 기준으로 사용한다.  
원본 데이터는 바뀌지 않고 순서만 결정된다.

- `-x[0]` : 재생수 내림차순 (Python sort는 기본 오름차순이므로 음수로 뒤집음)
- `x[1]` : 고유번호 오름차순 (2순위)
- 튜플은 앞 원소부터 순서대로 비교하므로 1순위/2순위가 자동으로 적용된다.

**Step 3.** 장르를 총합 기준 내림차순으로 정렬한다.

```python
sumDict = sorted(sumDict.keys(), key=lambda x: sumDict[x], reverse=True)
```

`sorted()`는 어떤 iterable에도 사용 가능하며 새 리스트를 반환한다.  
딕셔너리에는 `.sort()`를 쓸 수 없으므로 반드시 `sorted()`를 사용해야 한다.

**Step 4.** 정렬된 장르 순서대로 순회하며 최대 2곡씩 뽑는다.

## 최종 코드

```python
def solution(genres, plays):
    songDict = {}

    for i in range(len(genres)):
        # 배운 점: list() 함수는 해당 값을 순회가능할 때(iterable)만 사용 가능하다. 숫자는 불가.
        # 따라서 []는 그 자체의 값을 리스트화 한다.
        # 배운 점: append() 함수는 None을 반환하기 때문에 =으로 받아서는 안 된다.
        # setdefault 함수는 딕셔너리의 해당 키가 없으면 오른쪽 값을 반환. 있으면 그대로 반환.
        songDict.setdefault(genres[i], []).append((plays[i], i))

    sumDict = {}

    for key, value in songDict.items():
        # 비교용 튜플을 새로 생성해서 진행한다. 첫번째 인덱스는 내림차순을 사용하기 위해 -를, 기본은 오름차순이니 아무것도 붙이지 않음.
        # 튜플에서는 비교할 때 앞 인덱스가 1순위가 됨
        songDict[key].sort(key=lambda x: (-x[0], x[1]))

        sumValue = 0
        for i in range(len(value)):
            sumValue += value[i][0]
        sumDict[key] = sumValue

    # 배운 점: 리스트는 sort() 함수가 사용 가능하지만, 나머지는 sorted()를 사용해야 함.
    # x에 sumDict.keys() 하나씩 넣고 sumDict[key]의 숫자값끼리 역순으로 sorted()
    sumDict = sorted(sumDict.keys(), key=lambda x: sumDict[x], reverse=True)

    answer = []

    # 장르 1개
    if len(sumDict) == 1:
        if len(songDict[sumDict[0]]) == 1:
            answer.append(songDict[sumDict[0]][0][1])
        else:
            answer.append(songDict[sumDict[0]][0][1])
            answer.append(songDict[sumDict[0]][1][1])

    # 장르 2개 이상 - 장르별로 최대 2곡씩 뽑아내야 함
    else:
        for i in range(len(sumDict)):
            if len(songDict[sumDict[i]]) == 1:
                answer.append(songDict[sumDict[i]][0][1])
            elif len(songDict[sumDict[i]]) >= 2:
                answer.append(songDict[sumDict[i]][0][1])
                answer.append(songDict[sumDict[i]][1][1])

    return answer
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n log n)` |
| 공간 복잡도 | `O(n)` |

> `n` = 노래 수 (최대 10,000)

장르별 정렬이 지배적이며, 전체 노래 수 기준 `O(n log n)` 안에서 처리된다.

## 배운 점

**문제 이해가 먼저다**  
코드 구조는 거의 맞았는데 "장르별 2곡"이 아닌 "전체 2곡"으로 잘못 이해해서 오랜 시간을 소비했다.  
구현 전에 예시 케이스를 직접 손으로 추적해보는 습관이 중요하다.

**튜플 정렬에서 `key=lambda x: (-x[0], x[1])` 패턴**  
Python sort는 항상 오름차순 기준이므로, 내림차순이 필요한 값에 `-`를 붙여 뒤집는 트릭을 사용한다.  
`key` 함수는 비교용 튜플을 임시로 생성할 뿐, 원본 데이터는 변경되지 않는다.  
튜플은 앞 원소부터 순서대로 비교하므로 1순위/2순위 기준을 자연스럽게 표현할 수 있다.

**`sorted()`와 `.sort()`의 차이**  
`.sort()`는 리스트 전용 in-place 정렬로 `None`을 반환한다.  
`sorted()`는 어떤 iterable에도 사용 가능하며 새 리스트를 반환한다.  
딕셔너리 정렬에는 반드시 `sorted()`를 사용해야 한다.

**`setdefault`로 딕셔너리 초기화를 단순화할 수 있다**  
`if/else`로 키 존재 여부를 판별하는 대신, `setdefault(key, [])`는 키가 없으면 초기화하고 있으면 그대로 반환하므로 한 줄로 처리 가능하다.

**`append()`는 `None`을 반환한다**  
`result = list.append(x)` 형태로 받으면 `result`는 `None`이 된다.  
반환값을 변수에 담지 않고 단독으로 호출해야 한다.

**`len(sumDict[0])`은 문자열 길이를 반환한다**  
`sumDict`가 장르 이름 문자열의 리스트일 때, `sumDict[0]`은 `'classic'` 같은 문자열이다.  
노래 수를 확인하려면 반드시 `len(songDict[sumDict[0]])`으로 접근해야 한다.

---

> 이 문제의 핵심은 정렬 기준이 **장르 단위**와 **곡 단위** 두 겹으로 존재한다는 것이다.  
> 각 단계에서 무엇을 기준으로 정렬할지를 명확히 정의하면, 구현은 자연스럽게 따라온다.
