## 문제 설명

전화번호부에 적힌 번호들 중에서,  
**한 번호가 다른 번호의 접두어(prefix)** 인 경우가 있는지 확인해야 한다.

예를 들어 다음과 같은 경우:

- `"119"`
- `"97674223"`
- `"1195524421"`

여기서 `"119"`는 `"1195524421"`의 앞부분과 같으므로 접두어이다.  
이처럼 어떤 번호가 다른 번호의 접두어이면 `false`,  
그렇지 않으면 `true`를 반환해야 한다.

| phone_book | 결과 |
|-----------|------|
| `["119", "97674223", "1195524421"]` | `false` |
| `["123", "456", "789"]` | `true` |
| `["12", "123", "1235", "567", "88"]` | `false` |

---

## 입출력

**입력**

- `phone_book`: 전화번호가 담긴 문자열 배열

**출력**

- 어떤 번호가 다른 번호의 접두어이면 `false`
- 그렇지 않으면 `true`

**제한사항**

- `phone_book`의 길이는 1 이상 1,000,000 이하
- 각 전화번호의 길이는 1 이상 20 이하
- 같은 전화번호가 중복해서 들어있지 않음

---

## 헷갈렸던 부분

### 1. 모든 번호를 다 비교하면 시간 초과가 날 수 있다

처음에는 각 번호를 다른 모든 번호와 비교하면 될 것처럼 보인다.

```python
for i in range(len(phone_book)):
    for j in range(i + 1, len(phone_book)):
        ...
```

하지만 이렇게 하면 최악의 경우 거의 모든 쌍을 확인해야 해서  
시간 복잡도가 매우 커진다. 전화번호 개수가 많기 때문에 단순 이중 반복 비교는 비효율적이다.

### 2. 정렬했는데도 뒤를 끝까지 볼 필요는 없다

정렬을 하면 비슷한 문자열끼리 가까이 모인다.  
즉, 어떤 번호가 다른 번호의 접두어라면 정렬 후에는 그 후보가 바로 옆에 붙게 된다.

예를 들어:

```
["12", "123", "1235", "567", "88"]
```

정렬된 상태에서는 `"12"`와 접두어 관계가 생길 수 있는 번호가  
가장 먼저 `"123"`처럼 바로 뒤에 나타난다.

### 3. 접두어 확인은 `startswith()`가 더 명확하다

`find()`로도 확인할 수 있다.

```python
phone_book[j].find(phone_book[i]) == 0
```

이 방법도 가능하지만, 확인하고 싶은 것은 정확히 "앞에서 시작하느냐" 이므로  
`startswith()`가 더 직관적이다.

```python
phone_book[i+1].startswith(phone_book[i])
```

## 풀이 접근

### ❌ 처음에 하기 쉬운 접근

정렬한 뒤, 현재 번호와 뒤의 번호들을 첫 글자가 같은 동안 계속 비교하는 방식으로 풀 수 있다.  
하지만 이 방식도 경우에 따라 비교 횟수가 많아질 수 있다.

### ✅ 핵심 아이디어 — 정렬 후 바로 다음 번호만 비교

문자열을 정렬하면 사전순으로 배치된다.  
이 상태에서는 접두어 관계가 있는 문자열들이 서로 인접하게 된다.

즉, `phone_book[i]`와 `phone_book[i+1]` 이 두 개만 비교해도 충분하다.  
`phone_book[i]`가 어떤 번호의 접두어라면, 그와 가장 비슷한 번호가 정렬 후 바로 뒤에 오기 때문이다.

**예시 1** — `["119", "97674223", "1195524421"]`

```
정렬 후: ["119", "1195524421", "97674223"]

"119"와 바로 다음 "1195524421"만 비교해도 접두어 관계를 바로 발견할 수 있다.
```

**예시 2** — `["12", "123", "1235", "567", "88"]`

```
"12"와 "123" 비교 → 접두어 관계 있음 → 바로 false
굳이 "1235"나 그 뒤까지 볼 필요가 없다.
```

---

## 최종 코드

```python
def solution(phone_book):
    phone_book.sort()
    for i in range(len(phone_book)):
        # 정렬했다면 그 다음 것만 봐도 충분
        # find() 함수도 사용 가능
        if i + 1 < len(phone_book) and phone_book[i+1].startswith(phone_book[i]):
            return False
    return True
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(n log n) |
| 공간 복잡도 | O(1) 또는 O(n) |

`n` = `phone_book`의 길이

`sort()`에 O(n log n), 이후 인접한 원소끼리 한 번씩만 비교하므로 O(n).  
따라서 전체 시간 복잡도는 **O(n log n)** 이다.

공간 복잡도는 정렬 방식에 따라 다르지만, 추가 자료구조를 거의 쓰지 않으므로 크게 늘어나지 않는다.

## 배운 점

- **문자열 접두어 문제는 정렬과 잘 어울린다.**  
  정렬하면 비슷한 문자열들이 가까이 모이기 때문에 비교 범위를 크게 줄일 수 있다.

- **정렬했다고 해서 뒤를 끝까지 볼 필요는 없다.**  
  접두어 후보가 인접한 위치에 오므로 바로 다음 원소만 보면 된다.

- **`startswith()`는 의도가 명확한 문자열 메서드다.**  
  `find() == 0`으로도 가능하지만, 접두어 판별에는 `startswith()`가 더 직관적이다.

- **시간 초과 문제는 "비교 범위를 줄일 수 있는 구조"를 찾는 게 중요하다.**  
  정렬이나 자료구조를 이용하면 완전 탐색처럼 보이는 문제를 훨씬 단순하게 바꿀 수 있다.

> 이 문제는 처음 보면 모든 번호를 서로 비교해야 할 것처럼 느껴진다.  
> 하지만 정렬을 활용하면 접두어 관계가 있는 번호들이 서로 붙게 되고,  
> 결국 인접한 번호끼리만 비교하면 되는 문제로 바뀐다.  
> 완전 탐색처럼 보이는 문제를 정렬로 단순화하는 좋은 예시였다.
