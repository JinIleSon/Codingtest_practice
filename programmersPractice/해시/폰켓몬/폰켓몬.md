## 문제 설명

연구실에 있는 폰켓몬 중 일부를 가져가려고 한다.  
전체 폰켓몬 수는 항상 짝수이며, 그중 정확히 절반인 `N/2`마리만 선택할 수 있다.

이때 목표는 **선택한 폰켓몬의 종류 수를 최대화하는 것**이다.

| nums | 선택 가능 수 | 최대 종류 수 |
|------|-------------|-------------|
| `[3, 1, 2, 3]` | 2 | 2 |
| `[3, 3, 3, 2, 2, 4]` | 3 | 3 |
| `[3, 3, 3, 2, 2, 2]` | 3 | 2 |

핵심은 많은 마리를 가져가는 것이 아니라,  
**서로 다른 종류를 최대한 많이 가져가는 것**이다.

---

## 입출력

**입력**

- `nums`: 폰켓몬의 종류 번호가 담긴 배열

**출력**

- 가져갈 수 있는 폰켓몬 종류 수의 최댓값

**제한사항**

- `nums`의 길이는 1 이상 10,000 이하의 짝수
- 폰켓몬의 종류 번호는 1 이상 200,000 이하의 자연수

---

## 헷갈렸던 부분

### 1. 비교해야 하는 값은 2개뿐이다

처음에는 어떤 폰켓몬을 골라야 할지 경우의 수를 따져야 할 것 같지만,  
실제로는 아래 두 값만 비교하면 된다.

```
1. 전체 폰켓몬의 서로 다른 종류 수
2. 내가 선택할 수 있는 마리 수 (N/2)

가져갈 수 있는 종류 수 = min(두 값 중 더 작은 값)
```

### 2. 종류 수는 set으로 바로 구할 수 있다

배열에는 같은 종류가 여러 번 들어 있을 수 있다.  
중요한 건 개별 마리 수가 아니라 서로 다른 종류의 개수다.

```python
nums = [3, 1, 2, 3]
set(nums)  # {1, 2, 3}  →  종류 수: 3
```

### 3. 종류가 많아도 다 가져갈 수는 없다

> 종류 수 < 선택 가능 수 → 종류 수만큼만  
> 선택 가능 수 < 종류 수 → 선택 가능 수만큼만

```python
# 따라서 정답은 항상:
min(종류 수, 선택 가능 수)
```

## 풀이 접근

### ❌ 처음에 놓치기 쉬운 생각

어떤 조합으로 고르면 가장 다양한 종류가 될지 직접 경우를 따져야 할 것처럼 느껴질 수 있다.  
하지만 이 문제는 조합을 실제로 만들 필요가 없다.

### ✅ 핵심 아이디어 — set으로 종류 수를 구하고 N/2와 비교

```
정답 = min(서로 다른 종류 수, 선택 가능한 마리 수)

# 서로 다른 종류 수   = len(set(nums))
# 선택 가능한 마리 수 = len(nums) // 2
```

**예시 1** — `[3, 3, 3, 2, 2, 4]`

```
종류 수      = {2, 3, 4} → 3
선택 가능 수 = 6 // 2   → 3
정답         = min(3, 3) = 3
```

**예시 2** — `[3, 3, 3, 2, 2, 2]`

```
종류 수      = {2, 3} → 2
선택 가능 수 = 6 // 2 → 3
정답         = min(2, 3) = 2
```

## 최종 코드

```python
def solution(nums):
    return min(len(set(nums)), len(nums) // 2)

    # 아래처럼 if문으로도 쓸 수 있지만, min()이 더 간결하다
    # numsSet = set(nums)
    # if len(numsSet) <= len(nums) // 2:
    #     return len(numsSet)
    # else:
    #     return len(nums) // 2
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(n) |
| 공간 복잡도 | O(n) |

`n` = `nums`의 길이

`set(nums)`를 만드는 과정에서 배열을 한 번 순회하므로 O(n)이 걸린다.  
이후 `len()`과 `min()`은 상수 시간에 처리된다.

---

## 배운 점

- **이 문제는 조합 문제가 아니라 개수 비교 문제였다.**  
  복잡하게 보지 않고 핵심 조건만 남기는 연습이 중요하다.

- **`set`은 중복 제거 문제에서 매우 강력하다.**  
  직접 세지 않고 `set(nums)` 하나로 종류 수를 바로 구할 수 있다.

- **`min()`으로 조건문을 더 간단히 줄일 수 있다.**  
  "더 작은 값 반환"이 목적이라면 `min()`이 `if/else`보다 더 직관적이다.

- **문제를 식으로 바꾸는 습관이 중요하다.**  
  `정답 = min(종류 수, N/2)`로 정리되는 순간 거의 끝이다.
