## 문제 설명

수포자 3인방은 각자 정해진 패턴으로 답을 찍는다. 정답 배열 `answers`가 주어졌을 때, 가장 많은 문제를 맞힌 사람을 오름차순으로 반환한다. 공동 1등이면 모두 포함한다.

| answers | return |
|---------|--------|
| `[1, 2, 3, 4, 5]` | `[1]` |
| `[1, 3, 2, 4, 2]` | `[1, 2, 3]` |

---

## 입출력

**입력**
- `answers` : 길이 1 이상 10,000 이하, 각 원소는 1~5

**출력**
- 가장 많이 맞힌 사람의 번호 배열 (오름차순)

---

## 헷갈렸던 부분

**패턴 인덱스를 어떻게 순환시키는가**

문제 번호 `i`가 패턴 길이를 넘어서도 계속 순환해야 한다.
처음에는 `i >= len` 조건으로 분기했는데, 사실 `i % len`은 `i < len`일 때도 동일하게 동작하므로 조건 없이 `i % len`만 써도 충분하다.

**공동 1등 처리를 어떻게 할 것인가**

단순히 최댓값 하나만 반환하면 되는 게 아니라, 동점자를 모두 포함해야 한다.
`(점수, 번호)` 튜플 리스트로 만들어 정렬한 뒤, 1등 점수와 같은 경우를 모두 골라내는 방식으로 처리했다.

---

## 풀이 접근

### 핵심 아이디어 — 패턴 순환 인덱스로 점수 집계 후 동점자 처리

```
핵심 원칙: 각 수포자의 패턴을 미리 배열로 정의하고,
i % 패턴길이로 인덱스를 순환시켜 정답과 비교한다.
정렬 후 최고점과 같은 점수를 가진 수포자를 모두 수집한다.
```

**Step 1.** 수포자별 찍기 패턴을 배열로 정의한다.

**Step 2.** `answers`를 순회하며 `i % 패턴길이`로 각 수포자의 해당 답을 구하고, 정답과 일치하면 카운트를 올린다.

**Step 3.** `(점수, 번호)` 튜플 리스트를 오름차순 정렬한다.

**Step 4.** 가장 높은 점수(`sorting[2][0]`)와 같은 점수를 가진 수포자를 모두 `result`에 담는다.

**Step 5.** `result`를 오름차순 정렬해 반환한다.

## 최종 코드

```python
def solution(answers):
    first = [1, 2, 3, 4, 5]
    second = [2, 1, 2, 3, 2, 4, 2, 5]
    third = [3, 3, 1, 1, 2, 2, 4, 4, 5, 5]

    firstLen = len(first)
    secondLen = len(second)
    thirdLen = len(third)

    countFirst = 0
    countSecond = 0
    countThird = 0

    for i in range(len(answers)):
        if i >= firstLen:
            firstIndex = i % firstLen
        else:
            firstIndex = i
        if i >= secondLen:
            secondIndex = i % secondLen
        else:
            secondIndex = i
        if i >= thirdLen:
            thirdIndex = i % thirdLen
        else:
            thirdIndex = i

        if first[firstIndex] == answers[i]:
            countFirst += 1
        if second[secondIndex] == answers[i]:
            countSecond += 1
        if third[thirdIndex] == answers[i]:
            countThird += 1

    sorting = [(countFirst, 1), (countSecond, 2), (countThird, 3)]
    sorting.sort(key=lambda x: x[0])

    result = [sorting[2][1]]

    if sorting[2][0] == sorting[1][0]:
        result.append(sorting[1][1])

    if sorting[2][0] == sorting[0][0]:
        result.append(sorting[0][1])

    result.sort()

    return result
```

**리팩토링 — `i % 패턴길이` 단순화**

`i >= len` 분기 없이 `i % len`만으로도 동일하게 동작한다.

```python
def solution(answers):
    first = [1, 2, 3, 4, 5]
    second = [2, 1, 2, 3, 2, 4, 2, 5]
    third = [3, 3, 1, 1, 2, 2, 4, 4, 5, 5]

    counts = [0, 0, 0]

    for i, ans in enumerate(answers):
        if first[i % len(first)] == ans:
            counts[0] += 1
        if second[i % len(second)] == ans:
            counts[1] += 1
        if third[i % len(third)] == ans:
            counts[2] += 1

    max_count = max(counts)
    return [i + 1 for i, c in enumerate(counts) if c == max_count]
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n)` |
| 공간 복잡도 | `O(1)` |

> `n` = answers의 길이 (최대 10,000)

`answers`를 한 번 순회하며 3명의 점수를 동시에 집계하므로 O(n). 패턴 배열과 결과 배열 크기는 고정이므로 공간은 O(1).

## 배운 점

**`i % len`은 조건 분기 없이 항상 쓸 수 있다**

`i < len`이면 `i % len == i`이므로, `i >= len` 조건 없이 처음부터 `i % len`만 써도 동일하게 동작한다.
순환 인덱스가 필요한 상황에서는 무조건 `i % 패턴길이`로 시작하는 것이 깔끔하다.

**`(값, 번호)` 튜플로 묶으면 정렬과 추출이 편해진다**

점수만 따로 관리하면 동점자 처리 시 번호를 다시 매핑해야 한다.
`(점수, 번호)` 튜플로 묶어두면 정렬 후에도 번호 정보가 보존되어 추출이 간단해진다.

**`key=lambda x: x[0]`의 의미**

튜플 리스트를 정렬할 때 `key=lambda x: x[0]`은 첫 번째 원소(점수)만을 기준으로 삼겠다는 뜻이다.
튜플은 기본적으로 앞 원소부터 순서대로 비교하므로, 이 경우 `sort()`만 써도 결과는 같다.
기준을 명시적으로 제어하고 싶을 때 `key=lambda`가 진가를 발휘한다.

---

> 이 문제의 핵심은 **패턴 순환을 `i % 패턴길이`로 표현**하고, **동점자를 빠짐없이 수집**하는 것이다.
> 완전탐색답게 요행 없이 모든 문제를 정직하게 비교하되, 인덱스 순환과 동점 처리를 깔끔하게 구현하는 것이 실력 차이가 난다.
