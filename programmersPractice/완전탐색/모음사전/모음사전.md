## 문제 설명

모음만으로 이루어진 사전은 "A"부터 시작해서 "UUUUU"로 끝난다.
단어를 추가할 때 한 글자를 더 붙이는 것을 먼저 시도하고, 더 이상 늘릴 수 없으면 마지막 글자를 다음 모음으로 바꾸는 방식으로 정렬된다.

| word | result |
|------|--------|
| `"AAAAE"` | `6` |
| `"AAAE"` | `10` |
| `"I"` | `1563` |
| `"EIO"` | `1189` |

---

## 입출력

**입력**
- `word` : 길이 1 이상 5 이하, 'A', 'E', 'I', 'O', 'U'로만 구성

**출력**
- 사전에서 word의 순서 (1-indexed)

---

## 헷갈렸던 부분

**사전 순서가 어떻게 되는가**

처음에는 알파벳 사전처럼 같은 길이끼리 정렬한다고 생각했다.
실제로는 깊이 우선으로 탐색하는 구조다.
"A" 다음은 "E"가 아니라 "AA"이고, "AAAAA"까지 다 채운 뒤에야 "AAAAE"로 넘어간다.

```
1:  A
2:  AA
3:  AAA
4:  AAAA
5:  AAAAA
6:  AAAAE
7:  AAAAI
...
10: AAAE
11: AAAEA
```

**visited를 set으로 쓰면 안 되는 이유**

처음에 `visited = set()`으로 작성했는데, set은 순서가 없어서 "AAAE" 같은 단어를 올바르게 만들 수 없다.
`visited = []`로 바꾸고 `''.join(visited)`로 현재 단어를 구성해야 한다.

**재귀 탈출 후 탐색이 계속되는 문제**

word를 찾고 `return count`를 해도 그 호출 하나만 끝날 뿐, 상위 for문은 계속 돈다.
`nonlocal find` 플래그를 두고 word를 찾으면 `find = True`로 설정한 뒤,
for문 진입 조건에 `find == False`를 걸어서 탐색을 중단시켰다.

**루트 노드 pop을 빠뜨린 문제**

dfs 내부에서는 자식 탐색 후 `visited.pop()`을 하는데,
solution의 for문에서 각 루트 노드(A/E/I/O/U)를 호출한 뒤 pop을 안 하면
다음 루트를 탐색할 때 visited에 이전 값이 남아있어서 단어 비교가 틀린다.
루트 호출 후에도 반드시 `visited.pop()`을 해야 한다.

## 풀이 접근

### 핵심 아이디어 — DFS로 트리를 순회하며 word 탐색

```
핵심 원칙: 사전 순서는 트리의 깊이 우선 순회와 동일하다.
각 노드를 방문할 때마다 count를 올리고,
현재 visited가 word와 같으면 탐색을 멈추고 count를 반환한다.
```

**Step 1.** 각 노드를 방문할 때 visited에 append하고 count를 올린다.

**Step 2.** `''.join(visited) == word`이면 `find = True`로 설정하고 탐색을 중단한다.

**Step 3.** `len(visited) < 5`이고 `find == False`인 경우에만 자식 노드(A/E/I/O/U)를 순회한다.

**Step 4.** 자식 노드를 모두 탐색한 뒤 `visited.pop()`으로 되돌린다.

**Step 5.** solution에서 루트 노드 A/E/I/O/U 각각에 대해 dfs를 호출하고, 호출 후에도 `visited.pop()`을 한다.

---

## 최종 코드

```python
def solution(word):
    count = 0
    visited = []
    vowel = ['A', 'E', 'I', 'O', 'U']
    find = False

    def dfs(v, visited):
        nonlocal count
        nonlocal word
        nonlocal find
        visited.append(v)
        # 방문 즉시 word와 비교
        if word == ''.join(visited):
            find = True
            return count
        # 길이 5 미만이고 아직 못 찾은 경우에만 자식 탐색
        if len(visited) < 5 and find == False:
            for node in vowel:
                if find == True:
                    break
                count += 1
                dfs(node, visited)
                # 자식 탐색 후 되돌리기
                visited.pop()
        return count

    # 루트 노드 A/E/I/O/U 각각 탐색
    for node in vowel:
        if find:
            break
        count += 1
        dfs(node, visited)
        # 루트 노드도 반드시 pop
        visited.pop()

    return count
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(5^5)` |
| 공간 복잡도 | `O(5)` |

> 사전에 담긴 전체 단어 수는 최대 5 + 5² + 5³ + 5⁴ + 5⁵ = 3905개로 고정.
> word를 찾으면 즉시 탐색을 멈추므로 최악의 경우에도 3905번 이하로 방문한다.
> visited는 최대 깊이 5까지만 쌓이므로 공간은 O(5) = O(1).

## 배운 점

**사전 순서는 알파벳 순이 아니라 깊이 우선 순서다**

같은 길이끼리 먼저 묶는 게 아니라, 한 글자를 붙일 수 있으면 먼저 붙이고 다 채운 뒤에 마지막 글자를 바꾼다.
이 패턴을 트리로 모델링하면 DFS 순회와 정확히 일치한다.

**nonlocal 변수는 재귀 깊이에 상관없이 공유된다**

`count`와 `find`를 nonlocal로 선언하면, 재귀가 몇 단계를 내려가든 같은 변수를 공유한다.
A 탐색이 끝난 count 값이 E 탐색에 그대로 이어지는 이유다.

**DFS에서 pop은 자식뿐 아니라 루트에도 필요하다**

dfs 내부에서 자식 탐색 후 pop을 하더라도, solution에서 루트 노드를 호출한 뒤 pop을 빠뜨리면
다음 루트 탐색 시 visited에 이전 루트가 남아있어 비교가 틀린다.
append한 모든 노드는 탐색이 끝난 뒤 반드시 pop해야 한다.

**재귀 탈출은 한 단계만 끝낸다**

`return`은 현재 호출 하나만 종료한다. 상위 for문을 멈추려면 별도의 플래그가 필요하다.
`find` 플래그로 탐색 중단 신호를 전파하는 패턴을 익혔다.

---

> 이 문제의 핵심은 **사전 순서를 트리의 DFS 순회로 모델링**하는 것이다.
> visited의 append/pop으로 현재 경로를 관리하고, nonlocal 플래그로 조기 탈출을 구현하는 것이 실력 차이가 난다.
