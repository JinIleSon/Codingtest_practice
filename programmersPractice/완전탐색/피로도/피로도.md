## 문제 설명

각 던전은 `[최소 필요 피로도, 소모 피로도]`로 구성된다.
현재 피로도가 최소 필요 피로도 이상일 때만 던전에 입장할 수 있고,
입장 후에는 소모 피로도만큼 피로도가 줄어든다.
유저의 초기 피로도 `k`와 던전 목록 `dungeons`가 주어질 때, 탐험 가능한 최대 던전 수를 반환한다.

| k | dungeons | return |
|---|----------|--------|
| `80` | `[[80,20],[50,40],[30,10]]` | `3` |

---

## 입출력

**입력**
- `k` : 1 이상 5,000 이하인 자연수
- `dungeons` : 세로 길이 1 이상 8 이하, 가로 길이 2인 2차원 배열

**출력**
- 탐험할 수 있는 최대 던전 수

---

## 핵심 관찰

탐험 순서에 따라 결과가 달라지기 때문에, 어떤 순서가 최적인지 규칙으로 미리 결정할 수 없다.

```
예시: k = 80, dungeons = [[80,20],[50,40],[30,10]]

순서 A → B → C: 80 → 60 → 20 → 세 번째 던전 입장 불가 (최소 30 필요) → 2개
순서 A → C → B: 80 → 60 → 50 → 세 번째 던전 입장 가능 → 3개
```

따라서 모든 탐험 순서(순열)를 시도해보고 최댓값을 구해야 한다.

던전의 최대 개수가 8개이므로 최악의 경우 `8! = 40,320`가지를 탐색한다.
컴퓨터 입장에서 40,320번 연산은 매우 작은 값이므로 완전탐색이 적합하다.

---

## 풀이 접근

### 핵심 아이디어 — 재귀 + 백트래킹으로 모든 순열 탐색

백트래킹을 사용하는 이유는 **모든 경우를 탐색해야 하는데, 선택을 취소하고 다른 것을 골라야** 하기 때문이다.

**Step 1.** `visited` 배열을 만들어 각 던전의 방문 여부를 관리한다.

**Step 2.** `backtracking(k, count)`를 재귀 호출하며 모든 순서를 탐색한다.
- for문에서 방문하지 않았고(`visited[i] == 0`) 피로도가 충분한(`k >= dungeons[i][0]`) 던전만 선택한다.
- 던전을 선택하면 `visited[i] = 1`로 표시하고, 바뀐 피로도와 count를 인자로 넘겨 재귀 호출한다.
- 재귀가 끝나면 `visited[i] = 0`으로 되돌린다. ← **백트래킹**

**Step 3.** 매 재귀 호출 시작마다 `result = max(result, count)`로 최댓값을 갱신한다.

> `k`와 `count`는 매개변수로 복사본을 넘기기 때문에 별도로 되돌릴 필요가 없다.
> `visited`는 공유 리스트이므로 직접 `visited[i] = 0`으로 되돌려야 한다.

## 최종 코드

```python
# 갈 수 있는 던전은 8개이기 때문에, 최악의 경우에 8!=40,320 밖에 되지 않는다. (백트래킹은 10!~12!까지 가능)
# 컴퓨터에서는 40,320는 작은 값이기 때문에 오히려 완전 탐색이 알맞다.
# 재귀 + 백트래킹 문제다. 이유는 모든 경우를 탐색해야 하는데, 선택을 취소하고 다른 걸 골라야 한다. (원하는 경우로 보기 위해 / 되돌리기 + 모든 경우 탐색)
# 동일한 for문으로 재귀를 진행해도 되는 이유는 visited[i] = 1인 부분은 다시 가지 않을 것이기 때문이다.

def solution(k, dungeons):
    count = 0
    result = 0

    # 방문한 곳을 1, 방문하지 않은 곳을 0으로 표기하기 위한 공간
    visited = []
    for _ in range(len(dungeons)):
        visited.append(0)

    def backtracking(k, count):
        # 변수 읽기나 리스트 변경은 nonlocal 필요 없지만, 변수 변경은 nonlocal 필요
        nonlocal result
        # 백트래킹에서 결과물은 for문 위에 두는 게 관례
        result = max(result, count)

        for i in range(len(dungeons)):
            # 방문하지 않았고 k가 충분하면
            if visited[i] == 0 and k >= dungeons[i][0]:
                visited[i] = 1
                # 재귀
                # 안으로 들어가면 피로도 및 count 변동. 변동되는 값은 별도로 지정하지 않고 인자에 변동사항을 넣고 가는 것이 깔끔함. 복사본으로 하기 때문에 추적이 쉬움(되돌려야 하는 값)
                backtracking(k - dungeons[i][1], count + 1)
                # 재귀로 인자를 넘기면 count를 되돌릴 필요 없음 - count 복사본이 들어갔기 때문
                visited[i] = 0

    backtracking(k, count)
    return result
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(N!)` |
| 공간 복잡도 | `O(N)` |

> 던전 수 N은 최대 8이므로 `8! = 40,320`번 탐색. visited 배열 크기도 N에 비례한다.

## 배운 점

**그리디로 풀 수 없는 이유**

탐험 순서에 따라 결과가 달라지기 때문에, "소모 피로도가 작은 것부터" 같은 고정 규칙으로는 최적을 보장할 수 없다.
제한 조건이 작을 때는 완전탐색이 가장 안전하고 확실한 방법이다.

**백트래킹 틀은 고정이다**

```
result 갱신  ← for문 위
for 선택지:
    if 조건:
        visited = 1
        재귀(변동값은 인자로)
        visited = 0  ← 되돌리기
```

조건문과 인자로 넘길 값만 문제에 맞게 바꾸면 된다.

**매개변수 복사본 vs 공유 변수**

- `k`, `count`처럼 **되돌려야 하는 값** → 인자로 넘기면 복사본이 전달되어 자동으로 되돌려진다.
- `visited`처럼 **공유해야 하는 리스트** → 직접 `visited[i] = 0`으로 되돌려야 한다.
- `result`처럼 **내부 함수에서 변경하는 일반 변수** → `nonlocal` 선언이 필요하다.

**백트래킹을 쓰는 신호**

- 숫자 제한이 유난히 작을 때 (N ≤ 10 수준)
- 최적 선택 규칙을 미리 정할 수 없을 때
- 모든 경우를 탐색해야 하고, 선택을 취소하며 다른 것을 골라야 할 때

---

> 이 문제의 핵심은 **던전 수가 최대 8개로 작기 때문에 완전탐색이 가능하고,
> 재귀 + 백트래킹으로 모든 탐험 순서를 시도하며 최댓값을 구하는** 것이다.
> 백트래킹 틀을 익혀두면 비슷한 완전탐색 문제에 그대로 적용할 수 있다.
