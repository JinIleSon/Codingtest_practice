## 문제 설명

n개의 송전탑이 전선을 통해 하나의 트리 형태로 연결되어 있다. 전선 중 하나를 끊어 두 개의 전력망으로 분할할 때, 두 전력망의 송전탑 개수 차이(절대값)를 최소화하여 반환한다.

| n | wires | result |
|---|-------|--------|
| 9 | `[[1,3],[2,3],[3,4],[4,5],[4,6],[4,7],[7,8],[7,9]]` | `3` |
| 4 | `[[1,2],[2,3],[3,4]]` | `0` |
| 7 | `[[1,2],[2,7],[3,7],[3,4],[4,5],[6,7]]` | `1` |

---

## 입출력

**입력**
- `n` : 2 이상 100 이하의 자연수 (송전탑 개수)
- `wires` : 길이가 n-1인 2차원 배열, 각 원소는 `[v1, v2]` (연결된 두 송전탑 번호)

**출력**
- 두 전력망의 송전탑 개수 차이의 최솟값

---

## 헷갈렸던 부분

**처음 접근 방향이 틀렸다**

처음에는 가장 많이 연결된 노드를 기준으로 탐색하려 했다. 하지만 최적의 전선이 반드시 많이 연결된 노드와 관련 있지 않아서, 방향 자체를 바꿔야 했다.

**DFS에서 visited가 공유된다는 것을 몰랐다**

재귀 호출로 DFS가 깊이 파고들 때, 각 호출이 같은 visited를 참조하고 있다. 그래서 어느 깊이에서 return해도 len(visited)는 동일하며, 의미있는 return은 최초 호출 하나뿐이다.

**visited 초기화 위치**

전선을 하나씩 끊어보는 반복문 바깥에 visited = []를 선언했다가 누적된 값이 섞여 오답이 나왔다. 전선을 끊을 때마다 visited를 초기화해야 한다.

**반대쪽 덩어리를 따로 구할 필요가 없다**

v1에서 DFS를 돌려 한쪽 개수 `a`를 구하면, 반대쪽은 `n - a`로 바로 구할 수 있다. 물리적으로 그래프를 나눌 필요가 없다.

---

## 풀이 접근

### 핵심 아이디어 — 완전탐색 + DFS

```
핵심 원칙: n ≤ 100이므로 전선을 하나씩 전부 끊어봐도 시간 내에 충분하다.
전선 하나를 끊은 뒤 DFS로 한쪽 덩어리의 크기를 구하고, 반대쪽은 n에서 빼면 된다.
```

**Step 1.** wires를 순회하며 양방향 인접 리스트(graph)를 구성한다.
DFS는 어느 노드에서 시작하든 모든 연결을 따라갈 수 있어야 하므로 양방향이 필수다.

**Step 2.** wires를 하나씩 순회하며 해당 전선을 graph에서 제거한다 (`remove`).

**Step 3.** v1을 시작점으로 DFS를 실행해 연결된 노드 개수 `a`를 구한다.
반대쪽은 `n - a`로 계산한다.

**Step 4.** `abs(a - (n - a)`가 현재까지의 최솟값보다 작으면 갱신한다.

**Step 5.** 전선을 다시 graph에 복원(`append`)하고 다음 전선으로 넘어간다.

**Step 6.** 모든 전선을 시도한 뒤 최솟값을 반환한다.

## 최종 코드

```python
from collections import defaultdict

def dfs(graph, v, visited):
    visited.append(v)
    for node in graph[v]:
        if node not in visited:
            dfs(graph, node, visited)
    return len(visited)

def solution(n, wires):
    graph = defaultdict(list)
    minValue = 999

    for i in range(len(wires)):
        # DFS는 양방향으로 값을 넣는 게 필수
        graph[wires[i][0]].append(wires[i][1])
        graph[wires[i][1]].append(wires[i][0])

    for i in range(len(wires)):
        visited = []

        # 전선 잘라내기
        graph[wires[i][0]].remove(wires[i][1])
        graph[wires[i][1]].remove(wires[i][0])

        # v1과 연결된 덩어리 크기를 DFS로 계산
        v1 = dfs(graph, wires[i][0], visited)
        v2 = n - v1

        if minValue > abs(v1 - v2):
            minValue = abs(v1 - v2)

        # 전선 원복
        graph[wires[i][0]].append(wires[i][1])
        graph[wires[i][1]].append(wires[i][0])

    return minValue
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n²)` |
| 공간 복잡도 | `O(n)` |

> `n` = 송전탑 개수 (최대 100)

전선은 n-1개이므로 최대 n-1번 끊어보고, 매번 DFS가 최대 n개 노드를 방문하므로 O(n²). n ≤ 100이므로 최대 약 10,000번 연산으로 충분하다.

---

## 배운 점

**입력값이 작으면 완전탐색을 먼저 고려한다**

n ≤ 100처럼 입력값이 작을 때는 모든 경우를 시도해도 시간 내에 통과한다. 복잡한 최적화보다 완전탐색이 더 단순하고 안전한 선택이다.

**DFS 틀은 외워두면 응용이 쉽다**

```python
def dfs(graph, v, visited):
    visited.append(v)
    for node in graph[v]:
        if node not in visited:
            dfs(graph, node, visited)
    return len(visited)
```

문제마다 바뀌는 건 visited에 담는 것과 return에서 반환하는 것뿐이다. 나머지 구조는 거의 고정이다.

**DFS와 백트래킹의 차이**

DFS는 한번 방문한 노드를 다시 가지 않는다. 백트래킹은 `visited.remove()`로 방문을 취소하고 다른 경로를 탐색한다. 이 문제는 연결된 노드를 전부 세기만 하면 되므로 DFS로 충분하다.

**graph는 양방향으로 구성해야 한다**

단방향으로 구성하면 특정 노드를 시작점으로 삼았을 때 연결된 노드로 이동하지 못하는 경우가 생긴다. 트리는 부모→자식 방향이 고정되지 않으므로 양방향이 필수다.

**반대쪽 덩어리는 n에서 빼면 된다**

한쪽 덩어리 크기 `a`를 구하면 반대쪽은 `n - a`다. graph를 물리적으로 분리하거나 DFS를 두 번 돌릴 필요가 없다.

---

> 이 문제의 핵심은 **입력값이 작으면 완전탐색**이라는 판단과, **전선을 하나씩 끊고 DFS로 덩어리 크기를 세는** 구조를 연결하는 것이다.
