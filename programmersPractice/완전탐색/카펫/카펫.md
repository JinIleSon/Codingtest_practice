## 문제 설명

중앙은 노란색, 테두리 1줄은 갈색으로 칠해진 격자 카펫이 있다.
갈색 격자 수 `brown`과 노란색 격자 수 `yellow`가 주어질 때, 카펫의 `[가로, 세로]` 크기를 반환한다.
가로 길이는 세로 길이와 같거나 더 길다.

| brown | yellow | return |
|-------|--------|--------|
| `10` | `2` | `[4, 3]` |
| `8` | `1` | `[3, 3]` |
| `24` | `24` | `[8, 6]` |

---

## 입출력

**입력**
- `brown` : 8 이상 5,000 이하인 자연수
- `yellow` : 1 이상 2,000,000 이하인 자연수

**출력**
- 카펫의 `[가로, 세로]` 크기 배열

---

## 핵심 관찰

카펫의 구조를 수식으로 표현하면 두 가지 조건이 성립한다.

```
1. width * length == yellow
2. (width+2) * (length+2) - yellow == brown
```

노란색 가로를 `width`, 세로를 `length`라 하면,
전체 카펫은 각 방향으로 2씩 커진 `(width+2) × (length+2)`이고
여기서 노란색 `yellow`를 빼면 갈색 수가 된다.

---

## 풀이 접근

### 핵심 아이디어 — yellow의 약수 쌍을 순회하며 조건 검증

**Step 1.** `width`를 1부터 yellow까지 1씩 증가시킨다.

**Step 2.** `yellow % width == 0`이면 `length = yellow // width`로 세로를 결정한다.

**Step 3.** 두 조건을 동시에 만족하고, `width >= length`(가로 ≥ 세로)가 되는 순간 break한다.
- `width * length == yellow`
- `(width+2) * (length+2) - yellow == brown`

**Step 4.** break 시점의 `width+2`, `length+2`를 반환한다.

> `width >= length` 조건에서 break하므로, 가로가 세로보다 짧아지기 전에 탐색을 종료한다.

---

## 최종 코드

```python
def solution(brown, yellow):

    # 노란색 영역의 가로
    width = 1

    # 노란색 영역의 세로
    length = 1

    while width <= yellow:
        if yellow % width == 0:
            length = yellow // width

        # brown 개수 = 전체 개수 - yellow 개수
        # width와 length가 곱해지면 yellow 크기여야 함
        # 가로 길이가 세로 길이를 넘어서면 break
        if brown == (width+2)*(length+2)-yellow and width * length == yellow and width >= length:
            break

        width += 1

    # 노란색 기준 가로·세로에 +2한 값이 전체 카펫 크기
    result = [width+2, length+2]

    return result
```

---

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(yellow)` |
| 공간 복잡도 | `O(1)` |

> `yellow` 최대 2,000,000이므로 최악의 경우 200만 번 반복이지만, 조건 만족 시 바로 break하므로 실제로는 훨씬 빠르게 종료된다.

---

## 배운 점

**전체 카펫 크기를 노란색 기준으로 역산할 수 있다**

가로·세로를 직접 탐색하려 하면 범위가 불분명해진다.
노란색 영역의 `width`, `length`를 기준으로 잡으면,
전체 카펫은 자연스럽게 `(width+2) × (length+2)`로 표현되고
brown 조건도 간단한 수식 하나로 검증할 수 있다.

**약수 쌍 순회로 곱이 일정한 두 수를 탐색할 수 있다**

`yellow = width * length`를 만족하는 쌍을 찾을 때,
`yellow % width == 0`이면 `length = yellow // width`로 결정된다.
이 패턴은 곱이 고정된 두 수를 탐색하는 문제에서 반복적으로 등장한다.

**가로 ≥ 세로 조건을 break 기준으로 활용할 수 있다**

문제 조건에 "가로 ≥ 세로"가 명시되어 있으므로,
`width >= length`가 되는 순간 정답을 찾은 것이다.
이 조건을 break 기준으로 삼으면 불필요한 탐색을 줄일 수 있다.

---

> 이 문제의 핵심은 **노란색 영역을 기준으로 전체 카펫 크기를 수식으로 표현하고,
> yellow의 약수 쌍을 순회하며 brown 조건을 동시에 검증**하는 것이다.
> 완전탐색이지만 조건을 수식으로 정리하면 탐색 범위와 종료 조건이 명확해진다.
