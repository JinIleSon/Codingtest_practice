## 문제 설명

각 명함은 가로/세로를 자유롭게 회전할 수 있다. 모든 명함을 수납할 수 있으면서 가장 작은 지갑의 넓이를 반환한다.

| sizes | return |
|-------|--------|
| `[[60, 50], [30, 70], [60, 30], [80, 40]]` | `4000` |
| `[[10, 7], [12, 3], [8, 15], [14, 7], [5, 15]]` | `120` |
| `[[14, 4], [19, 6], [6, 16], [18, 7], [7, 11]]` | `133` |

---

## 입출력

**입력**
- `sizes` : 길이 1 이상 10,000 이하, 각 명함의 [가로, 세로] 배열
- `w`, `h` : 1 이상 1,000 이하인 자연수

**출력**
- 모든 명함을 수납 가능한 가장 작은 지갑의 넓이 (정수)

---

## 헷갈렸던 부분

**회전의 의미를 어떻게 코드로 표현하는가**

명함을 "회전할 수 있다"는 것을 처음엔 막연하게 생각했다.
핵심은 각 명함에서 **더 작은 수를 항상 같은 방향(가로)으로, 더 큰 수를 세로로** 맞추면
모든 명함의 방향이 통일되어 단순히 각 축의 최댓값을 구하면 된다는 것이다.

**왜 작은 수를 앞으로 몰아야 하는가**

더 큰 수끼리만 세로 방향에서 경쟁하게 하면 세로의 최댓값이 최소화된다.
반대로 가로는 더 작은 수들끼리만 경쟁하므로 가로 최댓값도 자연스럽게 최소화된다.
즉, **두 축을 분리해서 각각 최솟값 내에서 최댓값을 구하는 구조**가 핵심이다.

---

## 풀이 접근

### 핵심 아이디어 — 내부 정렬로 방향 통일 후 각 축 최댓값

```
핵심 원칙: 모든 명함에서 작은 수를 가로, 큰 수를 세로로 맞추면
지갑의 가로는 max(작은 수들), 세로는 max(큰 수들)이 된다.
```

**Step 1.** 각 명함 내부를 `sort()`로 정렬한다.
- 작은 수가 `sizes[i][0]`, 큰 수가 `sizes[i][1]`로 통일된다.

**Step 2.** 모든 명함을 순회하며 `sizes[i][0]`의 최댓값(`max0`)과 `sizes[i][1]`의 최댓값(`max1`)을 구한다.

**Step 3.** `max0 * max1`을 반환한다.

---

## 최종 코드

```python
def solution(sizes):
    # 내부 리스트 sort. 작은 수가 [0], 큰 수가 [1]로 통일
    for item in sizes:
        item.sort()

    max0 = 0
    max1 = 0

    for i in range(len(sizes)):
        if max0 < sizes[i][0]:
            max0 = sizes[i][0]
        if max1 < sizes[i][1]:
            max1 = sizes[i][1]

    return max0 * max1
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n)` |
| 공간 복잡도 | `O(1)` |

> `n` = sizes의 길이 (최대 10,000)

각 명함 내부는 길이 2 고정이므로 `item.sort()`는 O(1). 전체 순회 O(n)이 지배한다.
`max0`, `max1` 두 변수만 추가하므로 공간은 O(1).

---

## 배운 점

**내부 리스트 정렬은 for문으로 각각 꺼내서 처리한다**

`data.sort()`는 외부 리스트 순서를 바꾸고, 내부 리스트 각각에 접근하려면 for문으로 꺼내서 `item.sort()`를 호출해야 한다.
"한 단계 더 들어가려면 for문으로 꺼낸다"는 감각이 2차원 배열 문제에서 반복적으로 쓰인다.

**`sort()`와 `[x[1], x[0]]`의 차이**

- `sorted(x)` → 값 기준, 작은 수가 앞으로
- `[x[1], x[0]]` → 위치 기준, 무조건 순서 뒤집기

이 문제에서는 "작은 수를 앞으로" 가 목적이므로 `sorted()` 또는 `item.sort()`가 정확하다.

**방향 통일 후 단순 최댓값 문제로 환원**

회전 가능하다는 조건을 처음에는 복잡하게 느꼈지만, 내부 정렬로 방향을 통일하면
결국 각 축의 최댓값을 구하는 단순한 문제로 바뀐다.
조건을 코드로 어떻게 표현할지 막힐 때는 "이 조건이 수학적으로 어떤 의미인지"부터 다시 생각해보는 것이 효과적이다.

> 이 문제의 핵심은 **명함 회전을 내부 정렬로 표현**하는 것이다.
> 작은 수를 가로, 큰 수를 세로로 통일한 뒤 각 축의 최댓값을 곱하면 최소 지갑 크기가 된다.
