## 문제 설명

맨 처음엔 모든 자리가 'A'로 채워져 있으며, 조이스틱으로 알파벳을 바꾸거나 커서를 이동해 목표 이름을 완성한다.

| 조작 | 동작 |
|------|------|
| ▲ | 다음 알파벳 (Z에서 위로 이동하면 A) |
| ▼ | 이전 알파벳 (A에서 아래로 이동하면 Z) |
| ◀ | 커서 왼쪽 이동 (첫 번째에서 이동하면 마지막으로) |
| ▶ | 커서 오른쪽 이동 (마지막에서 이동하면 첫 번째로) |

| name | return |
|------|--------|
| `"JEROEN"` | 56 |
| `"JAN"` | 23 |

---

## 입출력

**입력**
- `name` : 완성해야 하는 알파벳 대문자 이름 (길이 1 이상 20 이하)

**출력**
- 조이스틱 조작 횟수의 최솟값

---

## 헷갈렸던 부분

**1. 알파벳 변환 방향 기준**

A(0)에서 Z(25)까지 총 25번 이동 가능하다.  
A에서 위로 올라가면 B~M까지 최대 12번, 아래로 내려가면 Z~N까지 최대 13번이다.  
따라서 목표 알파벳이 M 이하이면 위로, N 이상이면 아래로(A→Z→...→N) 이동하는 게 이득이다.

**2. 아래로 내려갈 때 wrap-around 처리**

목표 알파벳이 N 이상이면 A에서 1씩 빼야 하는데, `chr(ord('A') - 1)`은 '@'가 된다.  
A에서 아래로 이동하면 Z가 되어야 하므로 별도 예외처리가 필요하다.

```python
if answer[i] == 'A':
    answer[i] = 'Z'
    count += 1
    continue
```

**3. 커서 이동을 매 순간 가장 가까운 쪽으로 선택하는 그리디가 최적이 아님**

처음에는 매 순간 왼쪽/오른쪽 중 가까운 쪽을 선택하는 그리디로 접근했으나, 이는 항상 최적해를 보장하지 않는다.  
연속된 A 구간이 존재할 때, 해당 구간을 건너뛰는 전략(오른쪽 갔다 왼쪽 wrap, 왼쪽 wrap 갔다 오른쪽)이 오른쪽으로 쭉 가는 것보다 이득일 수 있다.  
커서 이동은 **세 가지 이동 패턴 중 최솟값**을 취해야 한다.

---

## 풀이 접근

### 핵심 아이디어 — 알파벳 변환과 커서 이동을 분리해서 계산

이 문제는 두 가지 최적화를 독립적으로 처리한다.

#### 1. 알파벳 변환 횟수

각 위치마다 목표 알파벳까지의 최소 이동 횟수를 합산한다.

```
목표 알파벳이 M 이하 → 위로 이동 (A에서 올라가기)
목표 알파벳이 N 이상 → 아래로 이동 (A에서 내려가기, Z를 거쳐서)
```

#### 2. 커서 이동 횟수

연속된 A 구간(변환 불필요한 구간)을 건너뛰는 세 가지 패턴 중 최솟값을 취한다.

```
1. 오른쪽으로 쭉: n - 1
2. i까지 오른쪽 갔다가 왼쪽 wrap으로 next부터 처리: i * 2 + (n - next)
3. 왼쪽 wrap으로 먼저 next부터 처리하고 오른쪽으로 i까지: (n - next) * 2 + i
```

여기서 `i`는 오른쪽으로 이동한 거리(방향 전환점), `next`는 `i` 이후 연속 A 구간이 끝나고 처음 등장하는 non-A 위치다.  
모든 `i`에 대해 반복하며 `move`의 최솟값을 갱신한다.

**Step 1.** 각 자리를 순회하며 목표 알파벳까지의 변환 횟수를 `count`에 누적한다.

**Step 2.** 모든 위치 `i`에 대해 `next`를 구하고, 세 가지 커서 이동 패턴의 최솟값을 `move`로 계산한다.

**Step 3.** `count + move`를 반환한다.

---

## 최종 코드

```python
def solution(name):
    count = 0

    answer = []
    nameList = []
    for i in range(len(name)):
        nameList.append(name[i])
        answer.append('A')

    i = 0
    while nameList != answer:
        # M 이하 → 위로 이동
        while nameList[i] != answer[i] and nameList[i] <= 'M':
            answer[i] = chr(ord(answer[i]) + 1)
            count += 1
        # N 이상 → 아래로 이동 (A → Z → ... → 목표)
        while nameList[i] != answer[i] and nameList[i] >= 'N':
            if answer[i] == 'A':
                answer[i] = 'Z'
                count += 1
                continue
            answer[i] = chr(ord(answer[i]) - 1)
            count += 1
        i += 1

    n = len(name)
    move = n - 1  # 오른쪽으로만 가는 경우

    for i in range(n):
        next = i + 1
        while n > next and name[next] == 'A':
            next += 1

        # 오른쪽 i까지 갔다가 왼쪽 wrap
        move = min(move, i * 2 + (n - next))
        # 왼쪽 wrap 먼저 갔다가 오른쪽 i까지
        move = min(move, (n - next) * 2 + i)

    count += move

    return count
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n²)` |
| 공간 복잡도 | `O(n)` |

> `n` = name의 길이 (최대 20)

알파벳 변환 순회 `O(n)`, 커서 이동 계산에서 각 `i`마다 `next` 탐색이 최대 `O(n)`이므로 전체 `O(n²)`.  
n이 최대 20이므로 성능 문제는 없다.

## 배운 점

**그리디가 항상 최적해를 보장하지는 않는다**  
매 순간 가장 가까운 미완성 문자로 이동하는 그리디는 연속 A 구간이 존재할 때 최적이 아닐 수 있다.  
커서 이동은 전체 경로 관점에서 세 가지 패턴을 비교해야 한다.

**두 문제를 분리해서 생각하면 복잡도가 낮아진다**  
알파벳 변환(상하 조작)과 커서 이동(좌우 조작)은 서로 독립적이다.  
각각 최적화하고 합산하면 된다.

**연속 A 구간을 기준으로 next를 잡는 아이디어**  
방향을 전환할 최적 지점은 항상 연속 A 구간 직전이다.  
모든 `i`에 대해 `next`를 구하고 최솟값을 취하면 어떤 패턴이든 커버된다.

**`n - next`가 wrap 포함 거리인 이유**  
0에서 왼쪽 wrap으로 `next` 위치에 도달하는 거리는 `0 → n-1 → ... → next`로 `n - next`칸이다.  
`n - 1 - next`가 아님에 주의한다.

---

> 이 문제의 핵심은 알파벳 변환과 커서 이동을 분리하는 것이다.  
> 커서 이동에서 단순 그리디가 아닌 세 가지 패턴(오른쪽 쭉, 오른쪽 후 왼쪽 wrap, 왼쪽 wrap 후 오른쪽)의 최솟값을 취해야 한다는 점이 이 문제의 핵심 난이도다.
