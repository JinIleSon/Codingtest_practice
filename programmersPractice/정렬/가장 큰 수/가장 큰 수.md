## 문제 설명

0 또는 양의 정수가 담긴 배열 `numbers`가 주어질 때, 순서를 재배치하여 이어 붙인 수 중 가장 큰 수를 문자열로 반환한다.

| numbers | return |
|---------|--------|
| `[6, 10, 2]` | `"6210"` |
| `[3, 30, 34, 5, 9]` | `"9534330"` |

---

## 입출력

**입력**
- `numbers` : 길이 1 이상 100,000 이하, 각 원소 0 이상 1,000 이하

**출력**
- 순서를 재배치하여 만들 수 있는 가장 큰 수 (문자열)

---

## 헷갈렸던 부분

**앞자리만 비교하면 안 된다**

처음에는 앞자리부터 차례로 비교하고, 같으면 그 다음 자릿수를 비교하는 방식을 생각했다.
하지만 이 방식은 30과 3처럼 자릿수가 다른 경우 로직이 복잡해지고, 결국 완전탐색처럼 시간이 오래 걸릴 수 있다.
핵심은 **두 수를 이어붙인 결과물 자체를 비교 기준**으로 삼는 것이다.

**기본 정렬로는 원하는 순서가 나오지 않는다**

숫자 크기 기준으로 정렬하면 30이 3보다 크기 때문에 앞에 오지만,
실제로는 3이 앞에 와야 "330"이 "303"보다 크다.
비교 기준이 숫자 크기가 아닌 이상, 커스텀 정렬이 필요하다는 걸 눈치채야 한다.

---

## 풀이 접근

### 핵심 아이디어 — 이어붙인 결과를 기준으로 커스텀 정렬

두 수 A, B를 비교할 때 "AB"와 "BA" 중 어느 쪽이 더 큰지를 기준으로 정렬한다.
이 비교 기준은 추이성을 만족하므로, sort에 그대로 위탁하면 전체 순서가 올바르게 결정된다.

```
핵심 원칙: 두 수를 직접 이어붙여 비교하면, 모든 자릿수 비교를 한 번에 끝낼 수 있다.
```

**Step 1.** `cmp_to_key`를 이용해 커스텀 비교 함수를 정의한다.

**Step 2.** 두 수 a, b를 문자열로 변환한 뒤 이어붙여 비교한다.
- `str(a) + str(b) > str(b) + str(a)` → a가 앞 → `-1` 반환 (순서 유지)
- `str(a) + str(b) < str(b) + str(a)` → b가 앞 → `1` 반환 (순서 교체)
- 같으면 `0` 반환

**Step 3.** 문자열 비교는 앞자리부터 아스키코드 순으로 이루어지며, 숫자 문자의 아스키코드는 숫자 크기와 일치하므로 이어붙인 문자열 비교가 올바르게 동작한다.

**Step 4.** 예외 처리: 정렬 후 첫 번째 원소가 `0`이면 전체가 0이므로 `"0"`을 반환한다.

**Step 5.** 정렬된 숫자를 순서대로 이어붙여 문자열로 반환한다.

## 최종 코드

```python
from functools import cmp_to_key

# sort에 사용될 cmp_to_key 내부에는 항상 2개의 인자만 가능함
def compare(a, b):
    # a가 앞에 있는 게 더 큰 경우
    # -1 리턴 시 sort()에는 A, B 그대로 둠
    if str(a) + str(b) > str(b) + str(a):
        return -1
    # b가 앞에 있는 게 더 큰 경우
    # 1 리턴 시 sort()에는 B, A로 바꿈
    elif str(a) + str(b) < str(b) + str(a):
        return 1
    # 둘이 같다고 표현될 경우
    # 0 리턴 시 sort()에는 그대로 둠
    else:
        return 0

def solution(numbers):

    numbers.sort(key=cmp_to_key(compare))

    # 예외상황
    # 모두 정렬된 후 맨 앞이 0이면 전체가 0이므로 "0" 리턴
    if numbers[0] == 0:
        return "0"

    result = ""

    for i in range(len(numbers)):
        result += str(numbers[i])

    return result
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n log n)` |
| 공간 복잡도 | `O(n)` |

> `n` = numbers의 길이 (최대 100,000)

커스텀 비교 함수를 사용하지만 sort 자체는 O(n log n)이고, 비교 함수 내부는 문자열 이어붙이기 + 비교로 O(1)에 가깝다.
완전탐색으로 모든 순열을 구하면 O(n!)이 되는데, 커스텀 정렬로 이를 O(n log n)으로 해결한 것이 핵심이다.

## 배운 점

**시간 복잡도 냄새가 커스텀 정렬의 단서다**

완전탐색으로 접근하면 시간이 오래 걸릴 것 같다는 느낌이 오면, 정렬로 풀 수 있는지를 의심해본다.
정렬이 떠올랐으면 "기준이 뭔지"를 고민하고, 기준이 특이하면 커스텀 정렬로 이어진다.

**cmp_to_key의 반환값 규칙**

비교 함수는 양수/음수/0을 반환하며, sort는 이를 기준으로 순서를 결정한다.
- 양수(1) → a를 뒤로 (b, a 순서로 교체)
- 음수(-1) → a를 앞으로 (a, b 순서 유지)
- 0 → 순서 그대로

**문자열 비교는 아스키코드 순이라 숫자에 그대로 적용된다**

`"610" > "106"`처럼 문자열끼리 부등호 비교가 가능하며, 숫자 문자의 아스키코드는 숫자 크기와 일치한다.
숫자를 자릿수별로 따로 뜯을 필요 없이 문자열로 변환한 뒤 이어붙여 비교하면 한 번에 끝난다.

**예외 처리는 return 타입까지 확인해야 한다**

모든 원소가 0인 경우 `0`이 아닌 `"0"`을 반환해야 한다.
문제 조건에 "문자열로 return"이라고 명시된 경우, 예외 처리에서도 타입을 맞춰야 한다.

---

> 이 문제의 핵심은 **기본 정렬 기준으로는 원하는 순서가 나오지 않는다는 것을 인지하고, 이어붙인 결과를 비교 기준으로 삼는 커스텀 정렬**을 적용하는 것이다.
> 완전탐색의 시간 복잡도 문제를 정렬로 해결하는 전형적인 패턴이므로, 유사한 문제에서도 바로 떠올릴 수 있도록 익혀두자.
