## 문제 설명

배열 `arr`가 주어질 때, 연속으로 나타나는 숫자는 하나만 남기고 전부 제거한다.  
제거 후 남은 수들은 원래 배열의 순서를 유지하여 반환해야 한다.

| arr | return |
|-----|--------|
| `[1, 1, 3, 3, 0, 1, 1]` | `[1, 3, 0, 1]` |
| `[4, 4, 4, 3, 3]` | `[4, 3]` |

---

## 입출력

**입력**
- `arr` : 0부터 9까지의 숫자로 이루어진 배열 (크기 1,000,000 이하)

**출력**
- 연속 중복을 제거한 배열

**제한사항**
- 배열 arr의 크기 : 1,000,000 이하의 자연수
- 배열 arr의 원소의 크기 : 0보다 크거나 같고 9보다 작거나 같은 정수

---

## 헷갈렸던 부분

**1. 인덱스 변수 `i`를 직접 관리한 것**

처음에는 `answer[i]`처럼 인덱스를 직접 들고 다니며 비교했다.  
스택은 맨 위(top)만 보는 자료구조이므로, `answer[-1]`로 접근하는 것이 의도에 맞다.  
`i`는 항상 `len(answer) - 1`과 동일했기 때문에 불필요한 변수였다.

**2. `if` 두 개가 독립적으로 실행되는 구조**

`answer`가 비어있을 때 첫 번째 `if`에서 `pop()`을 하고 나서,  
`arr`가 비어버리면 두 번째 `if`의 `arr[-1]`에서 `IndexError`가 발생한다.  
`if/elif/else` 구조로 분기를 연결하거나 `continue`를 사용해야 한다.

**3. 빈 리스트 체크를 `len() == 0`으로 한 것**

파이썬에서 빈 리스트는 `False`로 평가된다(falsy).  
`len(answer) == 0` 대신 `not answer`로 쓰는 것이 파이썬 관용 표현이다.

## 풀이 접근

### 핵심 아이디어 — 스택의 top만 비교

이 문제는 "가장 최근에 확정된 값"과 현재 값을 비교하는 구조다.  
`answer`를 스택처럼 사용하면 `answer[-1]`이 항상 top 역할을 한다.

```
스택/큐의 핵심 원칙: 인덱스에 직접 접근하지 않는다.
→ 스택은 top(-1)만, 큐는 front(0)만 본다.
```

**Step 1.** `arr`를 뒤에서부터 `pop()`으로 꺼낸다.

**Step 2.** `answer`가 비어있으면 바로 `append()`한다.

**Step 3.** `answer[-1]`과 꺼낸 값이 같으면 버리고, 다르면 `append()`한다.

**Step 4.** 뒤에서부터 꺼냈으므로 마지막에 `[::-1]`로 역순 반환한다.

---

## 최종 코드

```python
def solution(arr):

    answer = []

    while len(arr) > 0:
        # answer에 아무것도 없을 때(빈 리스트는 False임)
        if not answer:
            answer.append(arr.pop())
        # 현재 answer[-1](마지막) 값이 넣었던 값과 동일한 값일 때
        if arr[-1] == answer[-1]:
            arr.pop()
        # 현재 answer[-1](마지막) 값이 넣었던 값과 다른 값일 때
        else:
            answer.append(arr.pop())

    return answer[::-1]
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n)` |
| 공간 복잡도 | `O(n)` |

> `n` = 배열 arr의 길이 (최대 1,000,000)

`pop()`은 O(1)이고 n번 반복하므로 전체 O(n).  
마지막 `[::-1]`도 O(n)이지만 전체 복잡도에 영향 없음.

---

## 배운 점

**스택/큐는 인덱스에 직접 접근하지 않는다**  
스택은 top(-1)만, 큐는 front(0)만 접근하는 것이 원칙이다.  
인덱스 변수를 직접 들고 다니는 순간, 스택이 아닌 배열을 쓰는 것과 다름없다.

**빈 리스트는 `False`로 평가된다 (falsy)**  
`len(answer) == 0` 대신 `not answer`가 파이썬 관용 표현이다.  
빈 리스트, 빈 문자열, 0, None 모두 falsy에 해당한다.

**`or` 연산자의 단락 평가 (short-circuit)**  
`if not answer or answer[-1] != val`에서 `not answer`가 True면 뒤는 실행하지 않는다.  
빈 리스트에서 `[-1]` 접근을 막는 안전장치로 활용할 수 있다.

**`pop()`은 O(1)이다**  
배열은 길이를 별도 변수로 관리하므로, 마지막 요소 접근은 `시작주소 + (size-1)`로 바로 계산된다.  
중간 삭제는 O(n)이지만, 끝에서의 삭제는 항상 O(1)이다.

**`[::-1]`의 비용**  
뒤에서부터 pop하면 마지막에 역순으로 뒤집어야 한다.  
앞에서부터 순회하며 `append`하면 이 과정이 불필요하다.  
단순 순서 유지 문제라면 for 순회 + append가 더 자연스럽다.

**스택/큐 사용 방식**
스택 → top을 기준으로 판단하고 push/pop 결정
큐 → front를 기준으로 판단하고 enqueue/dequeue 결정
---

> 이 문제의 핵심은 "가장 최근에 확정된 값"과 현재 값만 비교하면 된다는 것이다.  
> 스택의 top(-1)만 보는 패턴이 이 구조와 정확히 맞아떨어진다.
