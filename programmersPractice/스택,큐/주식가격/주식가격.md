## 문제 설명

초 단위로 기록된 주식가격이 담긴 배열 `prices`가 주어질 때, 각 시점에서 가격이 떨어지지 않은 기간(초)을 담은 배열을 반환한다.

| prices | return |
|--------|--------|
| `[1, 2, 3, 2, 3]` | `[4, 3, 1, 1, 0]` |

---

## 입출력

**입력**
- `prices` : 각 가격은 1 이상 10,000 이하인 자연수 (길이 2 이상 100,000 이하)

**출력**
- 각 시점에서 가격이 떨어지지 않은 기간(초)을 담은 배열

---

## 헷갈렸던 부분

**1. 매 루프마다 time 리스트 전체를 갱신하면 O(n²)가 된다**

처음에는 스택에 쌓인 요소들의 경과 시간을 매 루프마다 `[x + 1 for x in time]`으로 갱신하려 했다.
하지만 이 방식은 바깥 루프 O(n) × 안쪽 컴프리헨션 O(n) = O(n²)이 된다.
해결책은 경과 시간을 직접 저장하는 게 아니라 **스택에 인덱스를 같이 저장**하고, 꺼낼 때 `현재 인덱스 - 저장된 인덱스`로 계산하는 것이다.

**2. 루프 종료 후 스택에 남은 요소를 처리해야 한다**

중간에 pop되지 않고 끝까지 남아있는 요소들은 while 조건에서 걸리지 않으므로 별도로 처리해야 한다.
이 요소들의 시간은 `len(prices) - 1 - 저장된 인덱스`로 계산할 수 있다.

**3. if 조건 없이 무조건 push해야 한다**

while로 현재값보다 큰 요소를 모두 pop한 뒤, 현재 요소는 `if` 조건 없이 **무조건 push**해야 한다.
현재 인덱스는 아직 가격이 떨어졌는지 알 수 없는 상태이기 때문이다.

---

## 풀이 접근

### 핵심 아이디어 — 인덱스를 튜플로 스택에 저장하고, 미리 만든 배열에 직접 기록

경과 시간을 실시간으로 갱신하는 대신, **스택에 `(인덱스, 가격)` 튜플을 저장**하고 꺼낼 때 시간을 계산한다.
결과는 `prices`와 같은 길이의 배열을 미리 만들어두고, **해당 인덱스 위치에 바로 기록**한다.

```
핵심 원칙: 스택 top과 현재 값만 비교한다.
→ 현재값이 top보다 작으면 pop하며 시간 기록
→ 현재값은 무조건 push
```

**Step 1.** `prices`와 같은 길이의 `time` 배열을 0으로 초기화한다.

**Step 2.** `prices`를 앞에서부터 순회하며 `(인덱스, 가격)` 튜플을 스택에 push한다.

**Step 3.** 현재 가격이 스택 top의 가격보다 작으면, pop하면서 `time[꺼낸 인덱스] = 현재 인덱스 - 꺼낸 인덱스`를 기록한다. 이를 조건이 만족하지 않을 때까지 반복한다.

**Step 4.** while이 끝난 후 현재 요소를 무조건 push한다.

**Step 5.** 루프가 끝난 후 스택에 남은 요소들은 `time[꺼낸 인덱스] = len(prices) - 1 - 꺼낸 인덱스`로 처리한다.

---

## 최종 코드

```python
def solution(prices):

    pricesTime = []
    time = []

    # 리턴할 값인 time 리스트에 prices만큼의 길이 추가
    for i in range(len(prices)):
        time.append(0)

    for i in range(len(prices)):
        if i == 0:
            pricesTime.append((i, prices[i]))
            continue
        # 현재값이 스택 top보다 작으면 pop하며 시간 기록
        while pricesTime and prices[i] < pricesTime[-1][1]:
            time[pricesTime[-1][0]] = i - pricesTime[-1][0]
            pricesTime.pop()

        # 배운 점: 현재 위치의 가격 및 시간은 무조건 push해야 한다.
        pricesTime.append((i, prices[i]))

    # 끝까지 남아있는 스택 처리
    for i in range(len(pricesTime)):
        time[pricesTime[i][0]] = len(prices) - 1 - pricesTime[i][0]

    return time
```

---

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n)` |
| 공간 복잡도 | `O(n)` |

> `n` = prices의 길이 (최대 100,000)

각 요소는 스택에 한 번 push되고 한 번 pop되므로 전체 순회는 O(n).
스택과 결과 배열 모두 최대 n개의 요소를 저장하므로 공간도 O(n).

---

## 배운 점

**경과 시간을 직접 저장하지 말고 인덱스를 저장한다**  
매 루프마다 시간을 갱신하면 O(n²)이 된다. 스택에 `(인덱스, 가격)` 튜플을 저장하면 꺼낼 때 O(1)로 계산할 수 있다.

**결과 배열을 미리 만들고 인덱스로 직접 접근한다**  
순서가 뒤섞여 pop되더라도 `time[해당 인덱스]`에 바로 기록하면 정렬 없이 올바른 위치에 값을 넣을 수 있다.

**스택에서 pop한 뒤에는 무조건 push한다**  
while 조건을 통과한 현재 요소는 아직 떨어진 시점이 결정되지 않았으므로, `if` 조건 없이 항상 스택에 넣어야 한다.

**루프 종료 후 스택 잔여 처리를 빠뜨리면 안 된다**  
끝까지 남아있는 요소들은 while에서 pop되지 않으므로 별도로 처리해야 한다. 이 요소들의 시간은 `len(prices) - 1 - 해당 인덱스`로 계산한다.

---

> 이 문제의 핵심은 **인덱스를 스택에 함께 저장해 시간을 O(1)로 계산**하고, **미리 만든 배열에 인덱스로 직접 기록**하는 패턴이다.  
> 중간에 pop되는 요소와 끝까지 남는 요소, 두 가지 경우를 모두 처리해야 완성된다.
