## 문제 설명

실행 대기 큐에 있는 프로세스들의 우선순위 배열 `priorities`와, 몇 번째로 실행되는지 알고 싶은 프로세스의 위치 `location`이 주어질 때, 해당 프로세스가 몇 번째로 실행되는지 반환한다.

**규칙**

1. 큐에서 프로세스 하나를 꺼낸다.
2. 큐에 대기 중인 프로세스 중 우선순위가 더 높은 프로세스가 있다면 꺼낸 프로세스를 다시 큐 뒤에 넣는다.
3. 그런 프로세스가 없다면 꺼낸 프로세스를 실행하고 종료한다.

| priorities | location | return |
|------------|----------|--------|
| `[2, 1, 3, 2]` | `2` | `1` |
| `[1, 1, 9, 1, 1, 1]` | `0` | `5` |

---

## 입출력

**입력**
- `priorities` : 프로세스 우선순위 배열 (길이 1 이상 100 이하, 원소 1 이상 9 이하)
- `location` : 실행 순서를 알고 싶은 프로세스의 인덱스 (0-based)

**출력**
- 해당 프로세스가 몇 번째로 실행되는지 (`int`)

---

## 헷갈렸던 부분

**1. 같은 우선순위 값을 가진 프로세스들을 구별하지 못했다**

처음에는 큐에 우선순위 값만 넣고, 실행 시점에 `priorities[0] == originalPrior[location]` 으로 비교했다.  
하지만 이 방식은 **값이 같다**는 것만 확인할 뿐, **내가 찾는 바로 그 프로세스인지**를 확인하지 못한다.  
예를 들어 `[2, 1, 3, 2]`에서 우선순위 `2`인 프로세스가 두 개 있으면, 어느 쪽이 실행됐는지 알 수 없다.

**해결 아이디어**  
처음 큐를 만들 때부터 각 프로세스에 **원래 인덱스를 함께 묶어** 튜플로 저장한다.  
꺼낼 때마다 `location`과 인덱스를 비교하면, 값이 중복돼도 정확하게 원하는 프로세스를 식별할 수 있다.

---

## 풀이 접근

### 핵심 아이디어 — 인덱스를 함께 묶어 고유하게 식별한다

우선순위 값만으로는 같은 값을 가진 프로세스들을 구별할 수 없다.  
`(우선순위, 원래 인덱스)` 형태의 튜플로 큐를 구성하면, 실행 시점에 `location`과 인덱스를 비교해 정확하게 추적할 수 있다.

```
큐의 핵심 원칙: 현재 최대 우선순위보다 낮으면 뒤로 보낸다.
→ 최대 우선순위면 실행 카운트 증가 후, location과 인덱스가 일치하면 count 반환
```

**Step 1.** `(우선순위, 원래 인덱스)` 튜플 형태로 deque를 구성한다.

**Step 2.** `priorities`를 오름차순 정렬해 `sortPrior`를 만든다. `sortPrior[-1]`이 현재 최대 우선순위다.

**Step 3.** 큐의 맨 앞 프로세스가 현재 최대 우선순위가 아니면 뒤로 보낸다.

**Step 4.** 최대 우선순위라면 실행 카운트를 증가시키고, 인덱스가 `location`과 일치하면 `count`를 반환한다.

**Step 5.** 일치하지 않으면 해당 프로세스를 제거하고 `sortPrior`에서도 최대값을 pop한다.

## 최종 코드

```python
from collections import deque

def solution(priorities, location):
    tuplePrior = deque()

    # 값과 원래 인덱스를 함께 묶어 튜플로 생성
    for i in range(len(priorities)):
        tuplePrior.append((priorities[i], i))

    # 배운 점: 정렬을 사용하면 크기의 순서를 알 수 있다. 크기의 정도를 알고 싶을 때 활용
    sortPrior = sorted(priorities)
    # [1, 2, 2, 3] 형태(오름차순) → sortPrior[-1]이 항상 현재 최대 우선순위

    # 실행 카운트 추적
    count = 0

    while tuplePrior:
        if tuplePrior[0][0] != sortPrior[-1]:
            # 현재 최대 우선순위가 아니면 뒤로 이동
            tuplePrior.append(tuplePrior.popleft())
        elif tuplePrior[0][0] == sortPrior[-1]:
            count += 1
            # 찾는 프로세스라면 실행 순서 반환
            if location == tuplePrior[0][1]:
                return count
            # 아니라면 실행 후 제거, 최대값도 갱신
            tuplePrior.popleft()
            sortPrior.pop()
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n²)` |
| 공간 복잡도 | `O(n)` |

> `n` = priorities의 길이 (최대 100)

최악의 경우 프로세스 하나가 실행될 때마다 나머지 프로세스를 모두 순회할 수 있으므로 O(n²).  
deque와 sortPrior 모두 최대 n개의 원소를 저장하므로 공간은 O(n).  
단, 제한사항에서 n ≤ 100이므로 실질적으로 문제없다.

---

## 배운 점

**위치 추적이 필요할 때는 인덱스를 함께 묶는다**  
값만으로 대상을 식별하지 못할 때, `(값, 원래 인덱스)` 튜플로 큐를 구성하면 중복 값이 있어도 정확하게 추적할 수 있다.  
이 패턴은 큐/스택 시뮬레이션에서 자주 등장하므로 기억해둘 것.

**정렬로 현재 최대값을 관리한다**  
`sorted(priorities)`로 오름차순 배열을 만들면 `[-1]`이 항상 현재 최대 우선순위가 된다.  
프로세스가 실행될 때마다 `pop()`으로 최대값을 갱신하면 별도의 탐색 없이 O(1)로 최대값을 알 수 있다.

**`elif`는 `else`로 바꿔도 동일하다**  
`if A` / `elif not A` 구조는 사실상 `if A` / `else` 와 같다.  
두 조건이 완전히 반대라면 `else`로 쓰는 것이 더 간결하다.

---

> 이 문제의 핵심은 **값이 같은 프로세스들을 어떻게 구별할 것인가**다.  
> 큐에 값만 넣으면 식별이 불가능하고, 원래 인덱스를 함께 묶으면 중복 없이 정확하게 추적할 수 있다.
