## 문제 설명

`(` 또는 `)` 로만 이루어진 문자열 `s`가 주어질 때, 올바른 괄호이면 `true`, 아니면 `false`를 반환한다.  
올바른 괄호란 `(`로 열렸으면 반드시 짝지어서 `)`로 닫혀야 하는 것을 의미한다.

| s | answer |
|---|--------|
| `"()()"` | `true` |
| `"(())()"` | `true` |
| `")()("` | `false` |
| `"(()("` | `false` |

---

## 입출력

**입력**
- `s` : `(` 또는 `)` 로만 이루어진 문자열 (길이 100,000 이하)

**출력**
- 올바른 괄호이면 `true`, 아니면 `false`

---

## 헷갈렸던 부분

**1. 순회가 끝난 후 스택에 남은 `(`를 확인해야 한다**

`)` 를 만날 때마다 스택이 비어있으면 `false`를 반환하지만, 순회가 끝난 뒤에도 스택에 `(`가 남아있으면 닫히지 않은 괄호가 있다는 뜻이다.  
`"(()("` 같은 케이스가 이에 해당하며, 마지막에 `len(answer) > 0`으로 반드시 확인해야 한다.

---

## 풀이 접근

### 핵심 아이디어 — 스택으로 top을 기준으로 판단

가장 최근에 열린 `(`와 현재 `)`가 짝이 맞는지를 확인하는 구조가 LIFO와 일치한다.  
`(`를 push하고, `)`를 만나면 top을 pop하는 패턴이 스택과 자연스럽게 맞아떨어진다.

```
스택의 핵심 원칙: top만 보고 판단한다.
→ ( 만나면 push, ) 만나면 스택이 비었는지 확인 후 pop
```

**Step 1.** 문자열을 앞에서부터 순회한다.

**Step 2.** `(`를 만나면 스택에 `push`한다.

**Step 3.** `)`를 만났을 때 스택이 비어있으면 짝이 없으므로 `false`를 반환한다.

**Step 4.** 스택이 비어있지 않으면 `pop`해서 짝을 맞춘다.

**Step 5.** 순회가 끝난 후 스택에 남은 `(`가 있으면 `false`, 없으면 `true`를 반환한다.

---

## 최종 코드

```python
def solution(s):
    answer = []
    for i in s:
        if i == '(':
            answer.append(i)
        else:
            # 스택에 남은 (가 없으면 False
            if len(answer) == 0:
                return False
            answer.pop()
    # 배운 점: 남은 (가 있어도 정상적으로 닫히지 않은 것 유의하기
    if len(answer) > 0:
        return False
    return True
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n)` |
| 공간 복잡도 | `O(n)` |

> `n` = 문자열 s의 길이 (최대 100,000)

문자열을 한 번 순회하므로 O(n).  
최악의 경우 `(`만 가득하면 스택에 n개까지 쌓일 수 있으므로 공간도 O(n).

---

## 배운 점

**이 문제가 스택인 이유**  
가장 최근에 열린 `(`와 현재 `)`가 짝이 맞아야 한다는 구조가 LIFO와 일치한다.  
top만 보고 판단하고 top에서 꺼내는 패턴이 스택과 정확히 맞아떨어진다.

**순회 후 스택 잔여 확인을 빠뜨리면 안 된다**  
`)`를 만날 때 스택이 비어있는지만 확인하면 `"(()("` 같은 케이스를 잡지 못한다.  
순회가 끝난 뒤 `len(answer) > 0`으로 닫히지 않은 `(`가 남아있는지 반드시 확인해야 한다.

**`not answer`와 `len(answer) == 0`은 동일하다**  
파이썬에서 빈 리스트는 falsy라 `if not answer`로 쓰는 것이 관용 표현이다.  
`len(answer) == 0`도 동작은 같지만 파이썬답지 않은 표현이다.

> 이 문제의 핵심은 **가장 최근에 열린 괄호부터 닫혀야 한다**는 구조다.  
> 순회 중 스택이 비었을 때와 순회 후 스택에 잔여가 있을 때, 두 가지 false 조건을 모두 잡아야 한다.
