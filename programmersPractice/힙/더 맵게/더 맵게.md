## 문제 설명

스코빌 지수가 담긴 배열 `scoville`과 원하는 스코빌 지수 `K`가 주어질 때, 모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 섞어야 하는 최소 횟수를 반환한다.

섞는 공식은 아래와 같다.

```
섞은 음식의 스코빌 지수 = 가장 맵지 않은 음식 + (두 번째로 맵지 않은 음식 * 2)
```

| scoville | K | return |
|----------|---|--------|
| `[1, 2, 3, 9, 10, 12]` | 7 | 2 |

---

## 입출력

**입력**
- `scoville` : 길이 2 이상 1,000,000 이하, 원소는 0 이상 1,000,000 이하
- `K` : 0 이상 1,000,000,000 이하

**출력**
- 모든 음식의 스코빌 지수를 K 이상으로 만들기 위한 최소 섞기 횟수
- 불가능한 경우 -1

---

## 헷갈렸던 부분

**1. heapify는 반환값이 없다 (None)**

처음에 `heap = heapq.heapify(scoville)`로 작성해서 `heap`이 `None`이 되는 문제가 발생했다.
`heapify`는 리스트를 제자리에서 힙으로 변환하므로, `scoville` 자체를 그대로 힙으로 사용해야 한다.

```python
# 잘못된 방법
heap = heapq.heapify(scoville)  # heap = None

# 올바른 방법
heapq.heapify(scoville)  # scoville 자체가 힙이 된다
```

**2. -1 리턴 조건의 위치**

`while`문 진입 후 `heappop` 하기 전에, 남은 음식이 1개인 경우를 체크해야 한다.
1개만 남은 상태에서 최솟값이 K 미만이면 더 이상 섞을 수 없으므로 -1을 반환한다.

**3. 최솟값끼리 섞는 것이 왜 최소 횟수인가**

K보다 이미 큰 값은 건드릴 필요가 없다. 중간값부터 섞으면 괜히 큰 값까지 소비하게 되어 횟수가 늘어난다.
heapq를 사용해 항상 가장 작은 두 값부터 처리하는 것 자체가 최소 횟수를 보장한다.

---

## 풀이 접근

### 핵심 아이디어 — heapq로 항상 최솟값 두 개를 꺼내 섞고, 결과를 다시 힙에 넣는다

매번 가장 작은 두 값을 꺼내야 하므로 `heapq`가 적합하다.
`heappop`은 항상 현재 최솟값을 꺼내고, 꺼낸 뒤 자동으로 다음 최솟값이 루트로 올라온다.

```
핵심 원칙: 항상 최솟값 두 개를 꺼내 섞고, 결과를 다시 힙에 push한다.
→ scoville[0](루트)이 K 미만인 동안 반복
→ 남은 음식이 1개인데 K 미만이면 -1 반환
```

**Step 1.** `heapq.heapify(scoville)`로 scoville을 최소 힙으로 변환한다.

**Step 2.** `scoville[0]`(최솟값)이 K 미만인 동안 while 루프를 반복한다.

**Step 3.** 루프 진입 시 남은 음식이 1개면 더 이상 섞을 수 없으므로 -1을 반환한다.

**Step 4.** 최솟값 두 개를 `heappop`으로 꺼내고, 공식에 따라 섞어 다시 `heappush`한다.

**Step 5.** 섞을 때마다 count를 1 증가시키고, 루프가 끝나면 count를 반환한다.

---

## 최종 코드

```python
import heapq

def solution(scoville, K):
    # heapq.heapify는 반환값이 없다.(None)
    heapq.heapify(scoville)

    count = 0

    while scoville[0] < K:
        # while문 진입 시 heappop하기 전에 길이가 1개일 때 바로 -1 리턴 해야 함
        if len(scoville) == 1:
            return -1
        n1 = heapq.heappop(scoville)
        n2 = heapq.heappop(scoville)
        result = n1 + (n2 * 2)
        heapq.heappush(scoville, result)
        count += 1

    return count
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n log n)` |
| 공간 복잡도 | `O(n)` |

> `n` = scoville의 길이 (최대 1,000,000)

`heapify`는 O(n), `heappush`/`heappop`은 각각 O(log n)이며 최대 n번 반복되므로 전체 O(n log n).
힙은 scoville 자체를 사용하므로 추가 공간은 O(1), 전체 공간은 O(n).

## 배운 점

**힙은 항상 최상위(루트)가 최솟값이다**  
`heappop()`을 호출하면 항상 그 시점의 최솟값을 꺼내고, 꺼낸 뒤 자동으로 다음 최솟값이 루트로 올라온다.  
즉, 실시간으로 최솟값을 계속 뽑아낼 수 있다.

**heapq도 push/pop 시 비용이 발생한다 — 하지만 O(log n)**  
힙은 트리 구조라서 루트까지의 경로(높이 = log n)만 훑으면 된다.  
`sorted()`처럼 전체를 정렬하지 않고 log n번 비교만 하므로, 값을 지속적으로 뽑아내야 할 때 훨씬 효율적이다.

**sorted vs heapq 선택 기준**

| 상황 | 추천 |
|------|------|
| 전체를 정렬된 순서로 봐야 함 | `sorted()` |
| K번째 원소 (K가 고정, 작음) | `heapq` |
| 값이 실시간으로 계속 추가됨 | `heapq` |
| 최솟값/최댓값만 반복해서 꺼냄 | `heapq` |
| 한 번만 정렬하면 끝 | `sorted()` |

**최댓값을 뽑아내는 방법 — 음수 트릭**  
`heapq`는 최소 힙만 지원하므로, 넣을 때 `-`를 붙이고 꺼낼 때 다시 `-`를 붙인다.

```python
heapq.heappush(heap, -5)
max_val = -heapq.heappop(heap)  # 5
```

**K번째 원소 뽑기 — K번 pop**  
K번 `heappop()`을 반복하면 K번째로 작은 값을 얻을 수 있다.  
또는 `heapq.nsmallest(k, arr)[-1]`을 사용할 수 있으며, 이 함수는 힙이 아닌 일반 리스트에도 바로 사용 가능하다.

**heapify는 반환값이 없다**  
`heap = heapq.heapify(list)`로 쓰면 `heap`이 `None`이 된다.  
`heapq.heapify(list)` 후 `list` 자체를 힙으로 사용해야 한다.

**다익스트라(최단경로)에도 heapq가 사용된다**  
"지금까지 발견한 경로 중 가장 짧은 노드부터 탐색"해야 하므로, 매 단계마다 최솟값을 꺼내야 한다.  
이것이 heapq를 다익스트라에서 사용하는 이유다.

---

> 이 문제의 핵심은 **heapq로 항상 최솟값 두 개를 꺼내 섞는 것이 곧 최소 횟수를 보장**한다는 점이다.  
> K보다 큰 값은 건드릴 필요가 없으므로, 작은 값부터 처리해야 불필요한 섞기 없이 최소 횟수로 끝낼 수 있다.  
> 남은 음식이 1개인데 K 미만인 경우와, `heapify`의 반환값이 없다는 점, 두 가지 엣지 케이스를 놓치지 않는 것이 중요하다.
