## 문제 설명

각 작업에 대해 `[요청 시각, 소요시간]`을 담은 2차원 배열 `jobs`가 주어질 때, 우선순위 디스크 컨트롤러가 처리했을 때 모든 요청 작업의 반환 시간의 평균의 정수부분을 반환한다.

우선순위는 **소요시간이 짧은 것 → 요청 시각이 빠른 것 → 작업 번호가 작은 것** 순이다.

반환 시간은 아래와 같이 정의한다.

```
반환 시간 = 작업 종료 시각 - 작업 요청 시각
         = (현재 시각 - 요청 시각) + 소요시간
```

| jobs | return |
|------|--------|
| `[[0, 3], [1, 9], [3, 5]]` | 8 |

---

## 입출력

**입력**
- `jobs` : 길이 1 이상 500 이하
- `jobs[i] = [s, l]` : s는 요청 시각(0 ≤ s ≤ 1,000), l은 소요시간(1 ≤ l ≤ 1,000)

**출력**
- 모든 요청 작업의 반환 시간의 평균의 정수부분

---

## 헷갈렸던 부분

**1. jobs 정렬만으로 충분하지 않은 이유**

처음에는 jobs를 한 번 정렬하면 힙 없이도 풀 수 있지 않을까 생각했다.
하지만 들어오는 순서는 요청 시각 기준인데, 처리하는 순서는 소요시간 기준이라 두 기준이 뒤바뀌어 있다.
대기 큐에 여러 작업이 쌓였을 때 소요시간이 짧은 걸 동적으로 골라야 하므로 힙이 필요하다.

```
jobs 정렬  →  "언제 대기 큐에 넣을지" (요청 시각 기준)
힙        →  "대기 큐에서 뭘 꺼낼지" (소요시간 기준)
```

만약 우선순위가 요청 시각 → 소요시간 순이었다면 정렬 한 번으로 끝났을 것이다.
소요시간이 더 높은 우선순위이기 때문에 힙이 필요한 구조로 유도된다.

**2. [요청 시각, 소요시간] → [소요시간, 요청 시각]으로 순서를 바꾼 이유**

힙은 튜플의 앞 요소부터 비교해서 자동 정렬한다.
우선순위가 소요시간 → 요청 시각 → 작업 번호 순이므로, 힙에 넣을 때 이 순서에 맞게 튜플을 구성해야 한다.

```python
heapq.heappush(waitQueue, (job[0], job[1], i))
# job[0] = 소요시간, job[1] = 요청시각, i = 작업번호
```

**3. deque와 heapq는 함께 사용 불가**

`heapq`는 리스트에만 적용된다. deque에는 힙 연산이 되지 않는다.
또한 `heappush`로 넣어야 힙 구조가 유지되어 `heappop`으로 최솟값을 보장할 수 있다.
`append`로 넣으면 힙 구조가 깨진다.

**4. currentTime이 멈추는 무한루프 케이스**

waitQueue도 비어있고, jobs는 남아있지만 아직 요청 시각이 안 된 경우,
currentTime을 앞으로 움직이지 않으면 while문이 무한루프에 빠진다.
이 경우 `currentTime += 1`로 시간을 강제로 진행시켜야 한다.

**5. sorted()는 결과를 반드시 변수에 담아야 한다**

처음에 `sorted(jobs, key=...)`만 작성해서 정렬이 안 되는 문제가 발생했다.
`sorted()`는 새로운 리스트를 반환하므로 `jobs = sorted(jobs, key=...)`로 받아야 한다.

## 풀이 접근

### 핵심 아이디어 — jobs를 요청 시각으로 정렬 후, 현재 시간 기준으로 대기 큐(힙)에 넣고 소요시간 최솟값부터 처리한다

```
핵심 원칙: jobs는 요청 시각 순으로 정렬, 대기 큐는 소요시간 기준 최소 힙
→ currentTime >= 요청 시각인 작업들을 힙에 push
→ 힙에서 heappop으로 소요시간 최솟값 꺼내 처리
→ waitQueue도 비고 jobs도 없으면 종료
→ waitQueue는 비었지만 jobs가 남아있으면 currentTime += 1로 시간 진행
```

**Step 1.** `jobs`를 요청 시각 기준으로 정렬한다. (문제에서 순서를 보장하지 않음)

**Step 2.** `[요청 시각, 소요시간]`을 힙 우선순위에 맞게 `[소요시간, 요청 시각]`으로 변환한다.

**Step 3.** `jobs`를 deque로 변환한다. (popleft로 앞에서부터 순서대로 꺼내기 위해)

**Step 4.** `jobs`나 `waitQueue`가 남아있는 동안 while 루프를 반복한다.

**Step 5.** `currentTime >= 요청 시각`인 작업들을 모두 힙에 push한다. (작업 번호 i도 함께)

**Step 6.** `waitQueue`가 비어있지 않으면 반환 시간을 누적하고, `currentTime`을 소요시간만큼 전진시킨 후 `heappop`으로 제거한다.

**Step 7.** `waitQueue`가 비어있고 `jobs`가 남아있으면 `currentTime += 1`로 시간을 진행한다.

**Step 8.** 전체 반환 시간 합계를 작업 수로 나눈 정수부분을 반환한다.

## 최종 코드

```python
from collections import deque
import heapq

def solution(jobs):
    # jobs들의 요청 시각 정렬이 필요할 것임(문제에서 요청 시각에 대한 순서를 보장하지 않음)
    jobs = sorted(jobs, key=lambda x: (x[0], x[1]))

    # [요청 시각, 소요시간]에서 [소요시간, 요청 시각]으로 바꿔야 함
    # 리스트 내부의 값들의 요소들을 역순으로 하는 법
    # jobs = [x[::-1] for x in jobs]

    # 리스트 내부의 값들이 3개 이상일 때 요소 하나하나 위치 조정하는 법
    jobs = [[x[1], x[0]] for x in jobs]

    jobs = deque(jobs)

    # 대기 큐(힙)
    waitQueue = []
    heapq.heapify(waitQueue)

    # 현재 위치(시간)
    currentTime = 0

    # 결과를 담을 변수
    result = 0

    i = 0
    # jobs나 waitQueue가 남아있는 경우 반복
    while jobs or waitQueue:

        # 현재위치(시각)이 jobs에 남아있는 요청 시각보다 작은 경우에 대기 큐에 계속 넣음
        while jobs and currentTime >= jobs[0][1]:

            # waitQueue에 넣을 요소를 popleft하면서 각각 꺼냄
            job = jobs.popleft()

            # 개별 요소를 각각 넣어 튜플 형식으로 변경함. tuple() 함수는 arg가 하나밖에 들어가지 않고,
            # ()로만 묶어 개별 요소들을 튜플 형식으로 다시 묶을 수 있다.
            # 내부 요소가 튜플이면 앞 요소부터 오름차순으로 낮은 것부터 차례로 알아서 정렬됨
            heapq.heappush(waitQueue, (job[0], job[1], i))

            # 아래처럼 heappush를 일반 append처럼 사용하면 안됨
            # waitQueue.heappush((job[0], job[1], i))

            i += 1

        # heappush한 데이터들은 [0]에 접근할 때 항상 최솟값임
        if waitQueue and currentTime >= waitQueue[0][1]:
            # 현재위치를 얻어냈으면 결과를 담음
            # result += result + a가 아니라 result += a와 result = result + a와 같음
            result += currentTime - waitQueue[0][1] + waitQueue[0][0]

            # 현재위치를 완료된 작업만큼 더함
            currentTime += waitQueue[0][0]

            # 완료된 큐를 pop시킴.
            # 여기서 heappop은 heap 내부의 최솟값만 뽑아냄(힙을 써야하는 이유)
            # deque랑 heap은 함께 사용 불가
            # heappush로 형태를 잡고 넣어야 heappop으로 뽑아낼 수 있음
            if waitQueue:
                heapq.heappop(waitQueue)

        # 대기 큐에 남아있는 작업이 없을 경우 현재위치(시간)을 1증가
        # 아직 넣지 않은 jobs들이 currentTime(현재 시간)과 요청시각이 맞물릴 경우 currentTime += 1 적용하지 않기
        if not waitQueue and jobs and currentTime < jobs[0][1]:
            currentTime += 1

    return result // i
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(n log n)` |
| 공간 복잡도 | `O(n)` |

> `n` = jobs의 길이 (최대 500)

`sorted()`는 O(n log n), `heappush`/`heappop`은 각각 O(log n)이며 최대 n번 반복되므로 전체 O(n log n).
waitQueue는 최대 n개의 원소를 가지므로 공간 복잡도는 O(n).

## 배운 점

**정렬과 힙의 역할을 분리해야 할 때가 있다**

들어오는 순서(요청 시각)와 처리하는 순서(소요시간)가 다를 때, 정렬 한 번으로는 해결이 안 된다.
jobs 정렬은 "언제 대기 큐에 넣을지"를 위한 것이고, 힙은 "대기 큐에서 뭘 꺼낼지"를 위한 것이다.
두 역할이 분리되어 있다는 걸 인식하는 것이 핵심이다.

**힙의 튜플 비교 원리**

힙에 튜플을 넣으면 앞 요소부터 순서대로 비교해서 자동으로 정렬된다.
따라서 우선순위 순서에 맞게 튜플을 구성하면 별도의 정렬 없이 힙이 알아서 처리해준다.

```python
# 소요시간 → 요청시각 → 작업번호 순으로 우선순위를 반영
heapq.heappush(waitQueue, (소요시간, 요청시각, 작업번호))
```

**heappush vs append**

| 구분 | 설명 |
|------|------|
| `append` | 그냥 뒤에 붙임, 힙 구조 유지 안 됨 |
| `heappush` | 넣으면서 힙 구조 유지, heappop 최솟값 보장 |

`heappop`을 쓰려면 반드시 `heappush`로 넣어야 한다. `heapify` 후 `append`를 써도 힙 구조가 깨진다.

**deque와 heapq는 함께 사용 불가**

`heapq`는 리스트에만 동작한다. 순서대로 접근해야 하는 jobs는 deque로 관리하고,
우선순위로 꺼내야 하는 waitQueue는 리스트 + heapq로 분리해서 사용해야 한다.

**currentTime이 멈추는 케이스를 반드시 처리해야 한다**

waitQueue가 비어있고 jobs가 남아있지만 아직 요청 시각이 안 된 경우,
아무것도 처리하지 않고 currentTime도 안 움직이면 무한루프에 빠진다.
이 경우 `currentTime += 1`로 시간을 강제로 진행시켜야 한다.

**heapify는 빈 리스트에 쓸 필요가 없다**

`heapq.heapify([])`는 의미가 없다. 빈 리스트로 시작하고 `heappush`로만 넣으면 자동으로 힙 구조가 유지된다.
`heapify`는 이미 값이 있는 리스트를 힙으로 변환할 때 사용한다.

**sorted vs sort**

| 구분 | 설명 |
|------|------|
| `sort()` | 원본 수정, 반환값 없음(None) |
| `sorted()` | 원본 유지, 새 리스트 반환 |

`sorted(jobs, ...)`의 결과를 변수에 담지 않으면 그냥 버려진다. 반드시 `jobs = sorted(jobs, ...)`로 받아야 한다.

---

> 이 문제의 핵심은 **"들어오는 순서(요청 시각)"와 "처리하는 순서(소요시간)"가 다르다**는 점을 파악하고,
> jobs 정렬과 힙을 각각의 역할에 맞게 분리해서 사용하는 것이다.
> currentTime이 멈추는 무한루프 케이스와, heappush/append 혼용 금지, 두 가지 엣지 케이스를 놓치지 않는 것이 중요하다.
