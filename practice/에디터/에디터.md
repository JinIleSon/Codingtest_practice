# BOJ 1406 - 에디터

## 풀이 방식

**스택 2개(left, right)를 활용한 커서 구현**

커서를 기준으로 왼쪽 스택(`l`)과 오른쪽 스택(`r`)으로 나누어 관리한다.  
`l`의 맨 끝이 곧 커서 위치를 의미하며, 별도의 인덱스 없이 `pop()`과 `append()`만으로 모든 명령어를 O(1)에 처리한다.

---

## 핵심 아이디어

```
문자열: a b c | d e f  (| = 커서)

l = [a, b, c]
r = [f, e, d]  ← 역순 저장 (맨 끝이 커서 바로 오른쪽)
```

| 명령어 | 연산 | 복잡도 |
|--------|------|--------|
| L | `r.append(l.pop())` | O(1) |
| D | `l.append(r.pop())` | O(1) |
| B | `l.pop()` | O(1) |
| P $ | `l.append($)` | O(1) |

---

## 시간초과 원인 및 개선

처음에는 문자열 슬라이싱으로 구현했으나 시간초과 발생

- **기존 방식**: 문자열 슬라이싱 → 매 연산마다 새 문자열 생성 **O(n)** → 전체 **O(n × M)**
- **개선 방식**: 스택 2개 → 모든 연산 **O(1)** → 전체 **O(n + M)**

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

string = input().strip()

l = []
for s in string:
    l.append(s)

r = []

M = int(input().strip())

for i in range(M):
    S = input().strip()
    if len(l) > 0 and S == 'L':
        r.append(l.pop())
    elif len(r) > 0 and S == 'D':
        l.append(r.pop())
    elif len(l) > 0 and S == 'B':
        l.pop()
    elif 'P' in S:
        l.append(S[2])

print(''.join(l + r[::-1]))
```
