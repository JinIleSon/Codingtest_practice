# BOJ 1874 - 스택 수열 풀이 회고

https://www.acmicpc.net/problem/1874

## 문제 요약
1~N을 순서대로 스택에 push/pop하여 주어진 수열을 만들 수 있는지 판단하고, 가능하면 +/-를 출력, 불가능하면 NO를 출력.

---

## 막혔던 부분

### 1. 불필요한 상태 추적
pop한 값을 나중에 다시 체크해야 할 것 같아서 `trash`, `division` 같은 변수를 만들었다.

근데 사실 스택은 **1부터 순서대로만 push**하기 때문에, top이 target보다 크면 target은 스택 깊숙이 눌려있거나 이미 pop된 것이므로 **그 순간 바로 NO**다. 과거를 추적할 필요가 없었다.

### 2. division 계산 오류
"어디서부터 push할지"를 `max(trash)`나 `diff[-1]`로 계산하려 했는데, 그냥 **"지금까지 push한 마지막 숫자 + 1"** 하나면 충분했다.

### 3. 예외처리로 덧대기
근본 설계가 틀렸는데 고치는 대신 예외처리를 계속 추가했다. 코드가 점점 길어지고 꼬였다.

> 막힐 때 "예외처리 추가"보다 "처음 설계가 맞나?"를 먼저 의심하자.

### 4. 스택이 비었을 때 push 못하는 버그
```python
if len(stack) > 0:
    if stack[-1] < target:  # 스택이 비어있으면 push를 건너뜀
```
스택이 비어있어도 target까지 push해야 하는데, `len(stack) > 0` 조건이 방해했다.

---

## 공부된 부분

### 1. 단락 평가 (Short-circuit Evaluation)
파이썬의 `and`, `or`는 결과가 확정되면 뒤를 보지 않는다.

- `or` → 앞이 `True`면 뒤 안 봄 (True 찾으면 끝)
- `and` → 앞이 `False`면 뒤 안 봄 (False 찾으면 끝)

```python
# 스택이 비어있어도 stack[-1]에 접근 안 함 (안전)
if len(stack) == 0 or stack[-1] < target:

# 스택이 비어있으면 stack[-1]에 접근 안 함 (안전)
if len(stack) > 0 and stack[-1] > target:
```

### 2. 스택의 핵심 특성
1부터 순서대로만 push하기 때문에 스택 안의 값은 **항상 아래부터 위로 오름차순**이다.

따라서 NO 판단 조건은 딱 하나:
> **top > target → NO**

### 3. 설계 먼저, 코드 나중
변수가 많아질수록 복잡도가 올라간다. "이 변수가 왜 필요한가?" 설명이 안 되면 없애는 게 맞다.

## 최종 코드

```python
import sys

input = sys.stdin.readline
n = int(input().strip())

target = 0
stack = []
result = []
maxValue = 0

for i in range(n):
    target = int(input().strip())
    if len(stack) == 0 or stack[-1] < target:
        for j in range(maxValue+1, target+1):
            stack.append(j)
            result.append('+')
            if maxValue < j:
                maxValue = j
    if len(stack) > 0 and stack[-1] == target:
        result.append('-')
        stack.pop()
    if len(stack) > 0 and stack[-1] > target:
        result.clear()
        result.append('NO')
        break

for i in range(len(result)):
    print(result[i])
```
