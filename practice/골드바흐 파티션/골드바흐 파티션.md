## 문제 설명

짝수 `N`을 두 소수 `a`, `b`의 합으로 나타낼 때, `(a, b)`와 `(b, a)`는 같은 파티션으로 본다.  
이 파티션의 총 개수를 출력한다.

예를 들어 `N = 10`이라면:

| 쌍 | 소수 여부 |
|----|----------|
| (3, 7) | ✅ |
| (5, 5) | ✅ |

→ 골드바흐 파티션 수: **2**

---

## 입출력

**입력**
- 첫째 줄: 테스트 케이스 수 `T` (1 ≤ T ≤ 100)
- 각 테스트 케이스: 짝수 `N` (2 < N ≤ 1,000,000)

**출력**
- 각 테스트 케이스마다 골드바흐 파티션의 수 출력

**예시**

| 입력 | 출력 |
|------|------|
| 6 | 1 |
| 8 | 1 |
| 10 | 2 |
| 12 | 1 |
| 100 | 6 |

---

## 핵심 아이디어

### 1. 소수 전처리를 한 번만 — 에라토스테네스의 체

테스트 케이스마다 소수를 다시 구하면 시간 초과가 발생한다.  
입력 중 **최댓값까지 소수 여부 배열을 한 번만 생성**하고, 모든 케이스에서 재사용한다.

```
입력 전체 저장 → 최댓값 파악 → 체 1회 생성 → 각 N에 재사용
```

### 2. 절반까지만 탐색 — 중복 제거

`(a, b)`와 `(b, a)`는 같은 쌍이므로, `a ≤ N // 2`인 범위만 탐색하면 된다.

```
a = 3, b = N - 3  →  a ≤ N // 2 이면 항상 a ≤ b
```

`N // 2`를 넘는 순간 `break`하면 탐색 범위를 절반으로 줄일 수 있다.

### 3. 소수인 값만 반복문에 — `realPrimeNumber` 리스트 활용

`a`를 `2`부터 `N//2`까지 전부 순회하면 합성수도 다 보게 된다.  
소수만 모아둔 `realPrimeNumber` 리스트를 만들어 순회하면 불필요한 반복을 건너뛸 수 있다.

```python
# a가 2부터 N//2까지 전부 순회 (느림)
for a in range(2, t // 2 + 1): ...

# 소수만 담긴 리스트 순회 (빠름)
for a in realPrimeNumber:
    if a > t // 2: break
    ...
```

### 4. `a`를 정하면 `b`는 자동 결정

`a + b = N` 이므로 `a`가 정해지면 `b = N - a`로 바로 결정된다.  
`b`가 소수인지만 `primeNumber[b]`로 즉시 확인하면 된다.

## 풀이 접근

### ❌ 초기 접근 — 2부터 N//2까지 전부 순회

```python
for a in range(2, t // 2 + 1):
    b = t - a
    if primeNumber[a] and primeNumber[b]:
        result += 1
```

이 방식은 `a`가 합성수일 때도 반복문이 돈다.  
`N`이 최대 1,000,000이고 테스트 케이스가 100개이면 최악의 경우 5천만 번 순회하게 되어 **시간 초과**가 발생한다.

### ✅ 최종 접근 — 소수 리스트만 순회

소수만 담긴 `realPrimeNumber`를 미리 만들어두고, 거기서만 `a`를 뽑는다.

```
1. 입력 전체를 testCase에 저장
2. 최댓값까지 에라토스테네스의 체 생성
3. 소수만 모은 realPrimeNumber 리스트 생성
4. 각 N에 대해 realPrimeNumber 순회
   - a > N // 2 이면 break
   - b = N - a 가 소수면 result += 1
5. result 출력
```

---

## 최종 코드

```python
import sys

input = sys.stdin.readline

T = int(input().strip())

testCase = []
for i in range(T):
    testCase.append(int(input().strip()))

# 에라토스테네스의 체 — 한 번만 생성
maxValue = max(testCase)
primeNumber = [True] * (maxValue + 1)

i = 2
while i * i <= maxValue:
    j = 2
    while i * j <= maxValue:
        primeNumber[i * j] = False
        j += 1
    i += 1

primeNumber[0] = False
primeNumber[1] = False

# 소수만 담은 리스트
realPrimeNumber = []
for i in range(2, len(primeNumber)):
    if primeNumber[i]:
        realPrimeNumber.append(i)

# 각 테스트 케이스 처리
for t in testCase:
    result = 0
    for a in realPrimeNumber:
        if a > t // 2:       # 절반을 넘으면 중복 — 탐색 종료
            break
        b = t - a
        if primeNumber[b]:   # b도 소수면 파티션 1개 추가
            result += 1
    print(result)
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 체 생성 | `O(N log log N)` |
| 소수 리스트 생성 | `O(N)` |
| 각 케이스 처리 | `O(π(N/2))` |
| 전체 | `O(N log log N + T × π(N/2))` |

> `N` = 최댓값 (최대 1,000,000), `T` = 테스트 케이스 수 (최대 100), `π(x)` = x 이하 소수 개수

- 체는 한 번만 생성하므로 테스트 케이스가 많아도 추가 비용이 없다.
- 각 케이스에서 소수만 순회하므로 합성수를 모두 보는 방식보다 훨씬 빠르다.
- `N = 1,000,000`일 때 소수 개수는 약 78,498개로, 절반인 약 39,000번만 순회한다.

## 배운 점

**소수 전처리와 탐색을 분리해야 한다**  
테스트 케이스마다 소수를 다시 구하면 중복 계산이 쌓인다.  
입력을 먼저 전부 받은 뒤 최댓값 기준으로 체를 한 번만 만드는 구조가 핵심이다.

**절반까지만 탐색하면 중복이 사라진다**  
`(a, b)`와 `(b, a)`가 같은 파티션이므로, `a ≤ N // 2` 조건으로 탐색 범위를 절반으로 줄일 수 있다.  
`break` 한 줄로 불필요한 순회 전체를 막을 수 있다.

**소수만 담은 리스트가 성능을 바꾼다**  
`range(2, N//2)`를 전부 순회하면 합성수도 포함된다.  
소수만 모아둔 리스트를 따로 만들면, 같은 결과를 훨씬 적은 반복으로 얻을 수 있다.

**`a`를 정하면 `b`는 계산으로 결정된다**  
`b = N - a` 라는 관계식 덕분에 `b`를 별도로 탐색할 필요가 없다.  
소수 여부 배열에서 `primeNumber[b]`를 인덱스로 즉시 확인하면 끝이다.

---

> 처음에는 소수인지 확인만 하면 된다고 생각해서 전체 범위를 순회했지만,  
> 소수 전처리 재사용, 절반 탐색, 소수 리스트 활용이라는 세 가지 최적화를 조합하고 나서야  
> 시간 제한 0.5초를 통과할 수 있는 구조가 만들어졌다.
