## 문제 설명

주어진 문자열이 올바른 괄호 문자열(VPS, Valid Parenthesis String)인지 판별하는 문제.

- `(` 와 `)` 의 짝이 정확히 맞아야 한다.
- 짝이 맞으면 `YES`, 맞지 않으면 `NO` 출력.

```
입력 예시     출력 예시
(())())    →  NO
(((()())   →  NO
(()())     →  YES
```

---

## 풀이 아이디어

스택의 push / pop 동작을 카운터 `N`으로 표현한다.

- `(` 를 만나면 `N += 1` → 스택에 쌓는 동작 (push)
- `)` 를 만나면 `N -= 1` → 스택에서 꺼내는 동작 (pop)
- 순회 도중 `N < 0` 이 되는 순간 → 스택이 비어있는데 pop을 시도한 것이므로 즉시 `NO`
- 순회가 끝난 뒤 `N == 0` 이면 `YES`, `N > 0` 이면 열린 괄호가 남은 것이므로 `NO`

---

## 제출 코드

```python
import sys

input = sys.stdin.readline
T = int(input().strip())

N = 0

for i in range(T):
    S = input().strip()
    for j in S:
        if j == '(':
            N += 1
        else:
            N -= 1
            if N < 0:
                print('NO')
                break
    if N == 0:
        print('YES')
    elif N > 0:
        print('NO')
    N = 0
```

---

## 판별 흐름 정리

| 상황 | 의미 | 결과 |
|------|------|------|
| 순회 중 `N < 0` | 스택이 빈 상태에서 `)` 등장 | `NO` 후 `break` |
| 순회 종료 후 `N == 0` | 모든 괄호가 정확히 짝을 이룸 | `YES` |
| 순회 종료 후 `N > 0` | 닫히지 않은 `(` 가 남아 있음 | `NO` |

---

## 설계 의도

`break` 이후 `N`은 음수 상태다. 바깥의 `if N == 0`과 `elif N > 0` 둘 다 해당하지 않으므로 추가 출력 없이 자연스럽게 넘어간다. 이는 의도된 설계로, 세 가지 경우가 겹치지 않고 깔끔하게 분리된다.

```
())  의 경우:
(  → N = 1
)  → N = 0
)  → N = -1  →  NO 출력 후 break
              →  if N == 0  : 해당 없음
              →  elif N > 0 : 해당 없음  (추가 출력 없음)
```
