# BOJ 9093 — 단어 뒤집기

> 🔗 [문제 링크](https://www.acmicpc.net/problem/9093) &nbsp;|&nbsp; 시간 제한: 1초 &nbsp;|&nbsp; 메모리 제한: 128 MB

---

## 문제 설명

문장이 주어졌을 때, **단어의 순서는 유지**하되 **각 단어를 뒤집어서** 출력하는 문제.

```
입력 예시           출력 예시
I am happy today → I ma yppah yadot
```

---

## 풀이 아이디어

1. 문자열 전체를 뒤집는다 → `S[::-1]`
2. 뒤집힌 문자열을 한 글자씩 순회하며 단어를 `new_s`에 누적한다.
3. 공백을 만나거나 문자열 끝에 도달하면 `new_list`에 단어를 추가한다.
4. 완성된 `new_list`를 다시 뒤집으면 단어 순서가 원래대로 복원된다.
5. 단어들을 공백으로 이어 출력한다.

> **핵심 원리**: 전체 문자열을 뒤집으면 각 단어도 뒤집히고 단어 순서도 뒤집힌다.  
> 이후 단어 배열을 다시 뒤집으면 단어 순서만 원복되어 원하는 결과를 얻을 수 있다.

---

## 제출 코드

```python
import sys

input = sys.stdin.readline

T = int(input().strip())

for i in range(T):
    S = input().strip()
    S_reversed = S[::-1]
    new_list = []
    new_s = ""
    length = 0

    for j in S_reversed:
        if j == ' ' or length == len(S_reversed):
            new_list.append(new_s)
            new_s = ""
            length += 1
        else:
            new_s += j
            length += 1
            if length == len(S_reversed):
                new_list.append(new_s)
                new_s = ""

    answer = new_list[::-1]
    for k in range(len(answer)):
        print(answer[k], end=' ')
    print()
```

## 아쉬웠던 점 & 개선 포인트

### 1. 조건 중복 제거 가능

```python
# 현재 코드
if j == ' ' or length == len(S_reversed):
    ...
```

`length == len(S_reversed)` 조건은 사실 없어도 된다.  
`else` 블록 안에서 이미 `length == len(S_reversed)`일 때 `new_list.append()`를 처리하고 있기 때문에 중복이 발생한다.

---

### 2. 더 간결하게 쓸 수 있는 방법

`split()`과 슬라이싱을 조합하면 같은 결과를 훨씬 간결하게 표현할 수 있다.

```python
import sys
input = sys.stdin.readline

T = int(input().strip())
for _ in range(T):
    words = input().split()
    print(' '.join(word[::-1] for word in words))
```

- `split()`으로 단어 분리 → 각 단어에 `[::-1]` 적용 → `join()`으로 합치기
- 직접 구현한 로직과 **동일한 동작**, 코드는 훨씬 짧아짐

## 배운 점

| 항목 | 내용 |
|------|------|
| 문자열 슬라이싱 | `S[::-1]`으로 문자열 전체를 O(n)에 뒤집을 수 있다 |
| 빠른 입력 | `sys.stdin.readline`을 사용해 입력 속도를 개선한다 |
| 단어 분리 | `split()`은 공백 기준으로 단어를 자동 분리한다 |
| 리스트 역순 | `list[::-1]`으로 리스트를 뒤집을 수 있다 |
