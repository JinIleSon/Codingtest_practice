## 문제 설명

-2진법은 부호 없는 이진수로 정수를 표현하는 방식이다.  
일반 2진법의 자릿값이 `1, 2, 4, 8, ...`인 것처럼, -2진법의 자릿값은 `1, -2, 4, -8, 16, ...`이다.

| 10진수 | -2진수 |
|--------|--------|
| 0 | `0` |
| 1 | `1` |
| 2 | `110` |
| 3 | `111` |
| -1 | `11` |
| -13 | `110111` |

음수와 양수 모두 부호 비트 없이 표현된다는 점이 일반 2진수와의 핵심 차이다.

---

## 입출력

**입력**
- 10진수 정수 N (`-2,000,000,000 ≤ N ≤ 2,000,000,000`)

**출력**
- -2진법으로 변환한 결과 (앞자리 0 없이)

**예시**

| 입력 | 출력 | 설명 |
|------|------|------|
| `-13` | `110111` | -8 + 4 + 0 - 2 + 1 = -13 |
| `0` | `0` | 그대로 출력 |
| `1` | `1` | |

## 헷갈렸던 부분

**1. Python에서 음수를 음수로 나눌 때 나머지 처리**

일반적인 진법 변환은 `n % base`로 나머지를 구하고 `n //= base`로 줄여나간다.  
그런데 `-2`로 나눌 때 Python의 나머지 연산은 기대와 다르게 동작한다.

예를 들어 `-101 % -2`는 `-1`이 나온다. 하지만 -2진수에서 자릿값은 반드시 `0` 또는 `1`이어야 한다.

| 경우 | `n % -2` (Python) | 필요한 처리 |
|------|-------------------|------------|
| 음수 홀수 (`-101`) | `-1` | `n -= 1` 후 `-2`로 나누고, 자릿값 `1` 기록 |
| 양수 홀수 (`101`) | `1` | 바로 `-2`로 나누고, 자릿값 `1` 기록 |
| 짝수 (모두) | `0` | 바로 `-2`로 나누고, 자릿값 `0` 기록 |

나머지가 `0` 또는 `1`이 되도록 직접 조건을 분기해야 했다.

**2. `n % -2 != 0` 조건으로 홀수 여부를 구분**

처음에는 `n % 2 != 0`으로 홀수를 판별하려 했지만,  
음수인 경우 Python의 모듈러 결과가 `-1`이 나와 조건이 맞지 않았다.  
`n % -2 != 0`으로 판별하면 음수/양수 관계없이 일관되게 홀수를 잡을 수 있었다.

**3. `n == 0`과 `n == 1`의 종료 조건**

`while n != 1`로 반복하면 `n`이 `0`이 되는 경우를 따로 처리해야 한다.  
반복이 끝난 뒤 `n == 1`이면 마지막 자릿값 `1`을 추가해야 하므로,  
루프 조건과 루프 이후 처리를 분리해서 작성했다.

## 풀이 접근

### ❌ 처음에 놓치기 쉬운 생각

일반적인 진법 변환처럼 `n % -2`로 나머지를 구하면 된다고 생각하기 쉽다.

하지만 Python에서 음수를 음수로 나누면 나머지가 음수로 나올 수 있어서,  
그대로 쓰면 자릿값에 `0` 또는 `1`이 아닌 값이 들어간다.

### ✅ 핵심 아이디어 — 나머지가 항상 0 또는 1이 되도록 홀짝 분기

나머지를 직접 `%`로 쓰는 대신, 홀수인지 여부를 먼저 확인한다.

```
1. n이 홀수라면 → 자릿값 1을 기록, n에서 1을 빼서 짝수로 만든 후 -2로 나눔
2. n이 짝수라면 → 자릿값 0을 기록, 그대로 -2로 나눔
3. n이 0이 되면 → 반복 종료
4. 반복 종료 후 n이 1이라면 → 자릿값 1을 추가
5. 수집한 자릿값 리스트를 역순으로 출력
```

일반 진법 변환과 구조는 동일하지만,  
**음수 나머지가 생기지 않도록 직접 홀짝을 분기**하는 것이 핵심이다.

## 최종 코드

```python
import sys

input = sys.stdin.readline

# 초점을 항상 나머지가 0 또는 1이 나오도록 해야 함
# 나눠야하는 수가 음수 + 홀수면 -1 더해서 -2로 나누기 진행 => -2로 나누면 나머지는 1이 남음. 예외로 구성

n = int(input().strip())

formation = []

while n != 1:
    if n == 0:
        formation.append(0)
        break
    if n % -2 != 0:
        formation.append(1)
        n -= 1
        n //= -2
    elif n % -2 == 0:
        formation.append(0)
        n //= -2

# 시작부터 n이 1이거나 끝까지 나눠졌을 때는 n이 1임
if n == 1:
    formation.append(1)

formation = formation[::-1]

for i in formation:
    print(i, end='')
```

## 복잡도 분析

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | `O(log N)` |
| 공간 복잡도 | `O(log N)` |

> `N` = 입력된 10진수의 절댓값

매 반복마다 `n`을 `-2`로 나누므로 반복 횟수는 `log₂|N|`에 비례한다.  
자릿값을 저장하는 리스트의 크기도 같은 비율로 증가한다.

## 배운 점

**음수를 음수로 나눌 때 Python 나머지 연산을 조심해야 한다**  
`-101 % -2`가 `-1`을 반환한다는 것을 직접 확인하며,  
진법 변환에서 나머지 연산을 무조건 믿으면 안 된다는 점을 배웠다.  
나머지가 원하는 범위 안에 있는지를 먼저 확인하는 습관이 필요하다.

**나머지를 쓰지 않고 홀짝으로 분기하는 발상이 핵심이었다**  
`% -2` 결과가 믿을 수 없다면, 직접 홀수/짝수를 판별해서 자릿값을 결정하면 된다.  
복잡해 보이는 음수 진법도 분기 조건 하나로 정리된다는 점이 인상적이었다.

**종료 조건을 꼼꼼하게 짜야 한다**  
`while n != 1`만으로는 `n == 0`인 경우를 잡지 못한다.  
종료 조건과 루프 이후 처리를 함께 설계하는 것이 중요하다.

**일반 진법 변환 코드를 그대로 쓰면 안 된다**  
기저가 음수인 진법은 양수 진법과 구조가 비슷하지만,  
음수 나머지 문제 때문에 코드를 그대로 재사용할 수 없다.  
익숙한 패턴이라도 기반 조건이 달라졌는지 먼저 확인해야 한다.

> Python의 나머지 연산이 음수에서 어떻게 동작하는지를 파악하는 것이 더 핵심이었다.  
> 언어의 세부 동작을 이해하지 않으면 익숙한 패턴도 틀릴 수 있다는 점을 체감했다.
