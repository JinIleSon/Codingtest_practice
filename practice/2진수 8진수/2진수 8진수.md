## 문제 설명

2진수로 이루어진 문자열이 주어졌을 때, 이를 8진수 문자열로 변환하여 출력하는 문제다.

8진수 한 자리는 2진수 3자리와 대응된다.

| 2진수 | 8진수 |
|-------|-------|
| 000   | 0     |
| 001   | 1     |
| 010   | 2     |
| 011   | 3     |
| 100   | 4     |
| 101   | 5     |
| 110   | 6     |
| 111   | 7     |

따라서 입력 문자열을 오른쪽부터 3자리씩 끊어서 처리하면 된다.

**예시:**

```
110011
→ 110 / 011
→ 6 / 3
→ 결과: 63
```

---

## 입출력

**입력**

첫째 줄에 2진수 문자열 1개가 주어진다.

**출력**

주어진 2진수를 8진수로 변환한 결과를 출력한다.

**예시**

| 입력   | 출력 |
|--------|------|
| 110011 | 63   |

## 헷갈렸던 부분

### 1. 오른쪽부터 3자리씩 끊어야 한다는 점

처음에는 왼쪽부터 3자리씩 잘라도 되나 생각할 수 있다.  
하지만 진법 변환은 가장 낮은 자리부터 묶이는 구조이기 때문에, 반드시 오른쪽 기준으로 3자리씩 나누어야 한다.

```
1010

왼쪽부터 자르면 → 101 / 0
오른쪽 기준으로 → 1 / 010

→ 1, 2
→ 결과: 12
```

### 2. `n[0::-1]` 과 `n[0:0:-1]` 은 다르다

```python
n[0::-1]    # 0번 인덱스를 포함해서 가져옴
n[0:0:-1]   # 아무것도 가져오지 않음
```

- `0::-1` → 시작 위치 0 **포함**
- `0:0:-1` → 끝 인덱스 0은 **포함하지 않음**

그래서 마지막 남은 1자리 또는 2자리를 처리할 때, `n[i-1::-1][::-1]` 형태가 정상적으로 동작한다.

### 3. `[::-1]` 과 `sort(reverse=True)` 는 완전히 다르다

```python
eightList[::-1]           # 현재 순서만 반대로 뒤집기
eightList.sort(reverse=True)  # 문자열 값을 비교해서 내림차순 정렬
```

문자열 정렬은 숫자 크기가 아니라 문자 비교(`'1' > '0'`)를 기준으로 한다.  
따라서 "묶음의 순서 복원"에는 `sort()`가 아니라 `[::-1]`을 써야 한다.

### 4. `%`, `//` 없이도 풀 수 있다는 점

이 문제는 정수 변환 후 8진수로 바꾸는 방식도 떠오르지만,  
입력 자체가 문자열이고 자릿수도 길 수 있기 때문에 문자열 슬라이싱만으로 충분히 처리 가능하다.

> 핵심은 계산이 아니라 **문자열을 어떻게 잘 끊고 복원하느냐**다.

## 풀이 접근

### ❌ 처음 생각 — 숫자로 바꿔서 계산

처음에는 2진수를 정수로 바꾼 뒤, `% 8`, `// 8`로 나누어 8진수로 변환하는 방식도 생각할 수 있다.

1. 2진수를 10진수로 바꿈
2. 8로 나눈 나머지를 저장
3. 8로 계속 나누며 반복
4. 역순 출력

하지만 이 문제는 입력이 문자열로 주어지고, 굳이 큰 수 연산으로 갈 필요 없이 문자열을 3자리씩 끊는 방식이 더 직접적이라고 판단했다.

### ✅ 최종 접근 — 문자열을 오른쪽부터 3자리씩 잘라 직접 매핑

1. 입력 문자열 `n`을 받는다.
2. 뒤에서부터 3자리씩 잘라 `eightList`에 저장한다.
3. 남은 1자리 또는 2자리도 마지막에 함께 저장한다.
4. `eightList`를 다시 뒤집어 원래 자리 순서로 복원한다.
5. 각 2진수 묶음을 대응되는 8진수 문자로 출력한다.

#### 1) 뒤에서부터 3자리씩 자르기

```python
i = len(n)

while i > 0:
    if i - 3 <= 0:
        eightList.append(n[i-1::-1][::-1])
        break
    eightList.append(n[i-1:i-4:-1][::-1])
    i -= 3
```

`n[i-1:i-4:-1][::-1]` 는 뒤에서부터 3글자를 가져온 뒤 다시 뒤집어서 원래 순서의 3자리 문자열로 만드는 방식이다.

예를 들어 `110011`이면 `011`, `110` 순으로 들어가고,  
마지막에 `eightList[::-1]`로 복원하면 `110`, `011`이 된다.

#### 2) 2진수 3자리와 8진수 1자리를 직접 대응

```python
for s in eightList[::-1]:
    if s == '0' or s == '00' or s == '000':
        print(0, end='')
    elif s == '1' or s == '001':
        print(1, end='')
    elif s == '10' or s == '010':
        print(2, end='')
    elif s == '11' or s == '011':
        print(3, end='')
    elif s == '100':
        print(4, end='')
    elif s == '101':
        print(5, end='')
    elif s == '110':
        print(6, end='')
    elif s == '111':
        print(7, end='')
```

앞쪽에 남은 묶음은 길이가 1 또는 2일 수 있으므로, 짧은 형태와 3자리 형태를 함께 처리했다.

### 전체 코드

```python
import sys

input = sys.stdin.readline

n = input().strip()

eightList = []

i = len(n)

while i > 0:
    # i - 3 == 0이 되는 순간 끝까지 값을 가져오지 못해서 <= 0으로 해야 함
    if i - 3 <= 0:
        # n[0::-1]은 인덱스 0을 가져오고 n[0:0:-1]은 아무것도 가져오지 않음
        eightList.append(n[i-1::-1][::-1])
        break
    eightList.append(n[i-1:i-4:-1][::-1])
    i -= 3

for s in eightList[::-1]:
    if s == '0' or s == '00' or s == '000':
        print(0, end='')
    elif s == '1' or s == '001':
        print(1, end='')
    elif s == '10' or s == '010':
        print(2, end='')
    elif s == '11' or s == '011':
        print(3, end='')
    elif s == '100':
        print(4, end='')
    elif s == '101':
        print(5, end='')
    elif s == '110':
        print(6, end='')
    elif s == '111':
        print(7, end='')
```

## 복잡도 분석

| 항목          | 복잡도 |
|---------------|--------|
| 3자리씩 자르기 | O(N)   |
| 각 묶음 변환   | O(N)   |
| **전체**      | **O(N)** |

> N = 입력 문자열 길이

입력을 한 번 훑으며 3자리씩 나누고, 다시 한 번 출력하므로 전체 시간복잡도는 O(N)이다.  
추가 저장 공간은 3자리 문자열들을 담는 리스트 정도이므로 공간복잡도도 O(N)이다.

## 배운 점

- **2진수 → 8진수는 3자리씩 끊으면 된다**는 것을 체감했다. 공식처럼 외우는 것이 아니라, 8진수 한 자리가 2진수 3자리와 정확히 대응된다는 점을 직접 구현하면서 이해하게 됐다.

- **슬라이싱에서 start, end, step의 의미**를 더 분명히 알게 됐다. 특히 `n[0::-1]` 과 `n[0:0:-1]` 의 차이는 직접 부딪혀 보지 않으면 헷갈리기 쉬운 부분이었다. "end는 포함되지 않는다"는 규칙을 확실히 이해하게 됐다.

- **리스트 뒤집기와 정렬은 전혀 다른 동작**이라는 점을 구분하게 됐다. `[::-1]`은 순서 반전이고, `sort(reverse=True)`는 값 비교 기반 정렬이다. 문자열 리스트에서는 이 차이가 특히 크게 드러난다.

- **문자열 문제는 계산보다 구조가 더 중요할 수 있다.** 이 문제는 숫자를 크게 계산하는 문제가 아니라, 문자열을 어떤 기준으로 묶고 복원할지를 정확히 잡는 것이 핵심이었다.

> 처음에는 단순 진법 변환처럼 보였지만, 실제로는 문자열을 오른쪽부터 3자리씩 끊고 슬라이싱 방향을 정확히 다루는 문제였다. 이번 풀이를 통해 슬라이싱, 리스트 순서 복원, 문자열 비교 방식까지 함께 정리할 수 있었다.
