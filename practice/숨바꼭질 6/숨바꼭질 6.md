## 문제 설명

수빈이는 현재 위치 `S`에 있고, 동생들은 각각 `A1, A2, ..., AN` 위치에 있다.

수빈이는 1초마다 다음 두 이동 중 하나를 할 수 있다.

- `X + D`
- `X - D`

즉, 한 번 움직일 때마다 항상 같은 거리 `D`만큼만 이동할 수 있다.

모든 동생을 찾기 위해서는, 수빈이의 현재 위치 `S`에서 각 동생 위치까지의 거리 차이가 모두 `D`의 배수여야 한다.  
따라서 가능한 `D`의 최댓값은 각 거리 차이들의 **최대공약수**가 된다.

**예시:**

- `S = 3`
- 동생 위치: `1, 7, 11`

거리 차이:

- `|3 - 1| = 2`
- `|3 - 7| = 4`
- `|3 - 11| = 8`

이들의 최대공약수는 `2`이므로 가능한 `D`의 최댓값은 `2`이다.

---

## 입출력

**입력**

- 첫째 줄에 동생의 수 `N`, 수빈이의 위치 `S`
- 둘째 줄에 동생들의 위치 `A1, A2, ..., AN`

**출력**

- 가능한 `D`의 최댓값 출력

**예시**

| 입력 | 출력 |
|------|------|
| `3 3`<br>`1 7 11` | `2` |
| `3 81`<br>`33 105 57` | `24` |

## 헷갈렸던 부분

### 1. 이 문제가 최솟값 문제처럼 보였던 점

처음에는 수빈이의 현재 위치에서 동생들까지 갈 수 있는 거리 중 가장 작은 값이 답이 아닐까 생각할 수 있다.

하지만 이 문제는 단순히 가장 가까운 동생까지의 거리를 구하는 문제가 아니다.  
수빈이는 항상 `D`만큼만 이동할 수 있기 때문에, 모든 동생 위치가 `S`를 기준으로 같은 간격 위에 놓여 있어야 한다.

즉 중요한 것은 **최소 거리**가 아니라, 모든 거리 차이를 공통으로 나눌 수 있는 **가장 큰 값**이다.

### 2. 왜 거리 차이들의 최대공약수를 구해야 하는지

각 동생 위치 `Ai`에 도달하려면 `|S - Ai|`가 `D`의 배수여야 한다.  
모든 동생을 찾으려면 이 조건이 전부 만족되어야 하므로, `D`는 모든 거리 차이들의 공약수여야 한다.

그리고 문제는 그중 **최댓값**을 요구하므로 최대공약수를 구하면 된다.

### 3. `math.gcd()`는 리스트를 한 번에 받지 못한다는 점

`math.gcd()`는 리스트 전체를 직접 받지 못한다.  
따라서 앞에서부터 최대공약수를 **누적해서 계산하는 방식**으로 처리해야 한다.

- `gcd(a, b)`를 구한 뒤
- 그 결과와 `c`의 gcd를 구하고
- 다시 그 결과와 `d`의 gcd를 구하는 방식

이렇게 누적해도 결과는 전체 최대공약수와 같다.

## 풀이 접근

### ❌ 처음에 놓치기 쉬운 생각

수빈이의 현재 위치에서 동생들까지의 거리 중 가장 작은 값을 답이라고 생각할 수 있다.

하지만 `D`는 단순한 거리값이 아니라, 수빈이가 이동할 때마다 사용하는 **공통 간격**이다.  
따라서 한 동생만 갈 수 있는 값이 아니라, 모든 동생에게 갈 수 있는 값이어야 한다.

### ✅ 핵심 아이디어 — 거리 차이들의 최대공약수 구하기

각 동생 위치에 대해 `|S - Ai|` 를 구한다. 이 값들은 수빈이가 각 동생에게 도달하기 위해 필요한 거리 차이이다.  
이제 이 거리 차이들을 전부 나눌 수 있는 가장 큰 값, 즉 **전체 최대공약수**를 구하면 답이 된다.

**풀이 순서:**

1. 수빈이 위치 `S`와 각 동생 위치 `Ai`의 차이를 절대값으로 구한다.
2. 그 차이들을 리스트에 저장한다.
3. 첫 두 값의 gcd를 구한다.
4. 세 번째 값부터는 앞에서 구한 gcd와 누적해서 다시 gcd를 구한다.
5. 최종적으로 남은 값이 가능한 `D`의 최댓값이다.

> 동생이 1명뿐이라면 최대공약수를 구할 필요가 없으므로, 그 동생과의 거리 차이 절대값을 그대로 출력하면 된다.

## 최종 코드

```python
import sys
import math

input = sys.stdin.readline

N, S = map(int, input().strip().split())
younger = list(map(int, input().strip().split()))

# math.gcd()는 리스트 형태로 넣지 못함. 따라서 gcd를 누적시키는 방식으로 진행
# N이 1일 때(1명일 때)는 최대공약수 의미가 없으니 절대값 그대로 출력
if N == 1:
    print(abs(S - younger[0]))
else:
    subtraction = []

    for i in range(N):
        subtraction.append(abs(S - younger[i]))

    # 2명을 먼저 누적 (2명만 있으면 여기서 끝)
    GCD = math.gcd(subtraction[0], subtraction[1])

    # 3명부터 누적해 나감
    if N >= 3:
        for i in range(2, N):
            GCD = math.gcd(GCD, subtraction[i])

    print(GCD)
```

## 복잡도 분석

| 항목 | 복잡도 |
|------|--------|
| 시간 복잡도 | O(N) |
| 공간 복잡도 | O(N) |

> N = 동생의 수

각 동생에 대해 한 번씩 거리 차이를 계산하고, 최대공약수도 한 번씩 누적 계산하므로 전체 시간 복잡도는 O(N)이다.  
거리 차이를 저장하는 리스트를 별도로 사용했기 때문에 공간 복잡도는 O(N)이다.

## 배운 점

- **이 문제의 핵심은 최소 거리 찾기가 아니라 공통 간격 찾기였다.** 처음에는 현재 위치에서 갈 수 있는 최소값처럼 느껴질 수 있다. 하지만 실제로는 모든 동생 위치를 만족시키는 공통 이동 거리 `D`를 구해야 하므로, 핵심은 공통 배수 관계를 보는 것이었다.

- **위치 차이를 절대값으로 바꾸면 문제를 단순하게 볼 수 있다.** 수빈이가 왼쪽으로 가든 오른쪽으로 가든 중요한 것은 방향이 아니라 거리 차이이다. `abs(S - Ai)`로 바꾸면 문제를 훨씬 단순하게 정리할 수 있었다.

- **`math.gcd()`는 리스트 전체를 직접 받지 못하므로 누적해서 계산해야 한다.** 대신 앞에서부터 gcd를 계속 누적하는 방식으로 전체 최대공약수를 구할 수 있었다.

- **gcd를 누적해도 전체 최대공약수가 된다는 점을 이해했다.** 앞에서 나온 gcd는 지금까지 본 수들의 공통 약수 중 가장 큰 값이므로, 그 값과 다음 수의 gcd를 구하면 전체 범위의 최대공약수가 유지된다.

> 이 문제는 이동 규칙을 그대로 따라가며 시뮬레이션하는 문제가 아니라, 각 위치 차이의 공통된 간격을 찾는 수학 문제로 바라보는 것이 핵심이었다. 특히 최대공약수를 리스트에 누적 적용하는 방식이 왜 성립하는지 이해한 점이 가장 큰 수확이었다.
